# docs/AGENTS.md — Documentation

## Scope

ADRs (`docs/adr/**`), specs (`docs/specs/**`), architecture docs (`docs/architecture/**`), audit reports (`docs/audit/**`), impl records (`docs/audit/impl-records/**`), block-development guides (`docs/block-development/**`), facts registry (`docs/facts/**`), planning artifacts (`docs/planning/**`), Chinese mirror docs (`docs/zh-CN/**`).

## Policy

- ADRs and specs are governed by ADR-042/043/044 frontmatter schemas. New ADRs require valid YAML frontmatter (`status`, `governs.files`, `governs.modules`, `maintainers`, etc.) per ADR-042 §5.
- **No silent ADR or spec edits.** Every edit to `docs/adr/**` or `docs/spec/**` requires Tier-2 approval per ADR-043 §3.3 (during the cascade, this means owner approval).
- Audit logs under `docs/audit/overrides.log`, `docs/audit/governance-changes.log`, `docs/audit/commit-log.jsonl` are **append-only** — never rewrite history.
- The facts file `docs/facts/generated.yaml` is auto-generated by `scripts/audit/generate_facts.py` — hand edits are rejected by CI.
- Doc-drift between ADR-cited surface and code MUST be caught before merge (`doc_drift.py` audit; ADR-042 §9).
- Chinese mirrors under `docs/zh-CN/` MUST be regenerated when the English source changes (root rubric R9).

## Routing

| Need | Where |
|---|---|
| Add an ADR | Open issue → `adr-router` skill → tier-2 approval → merge |
| Add a spec | `/speckit.specify` then `/speckit.plan` |
| Add an impl record | `docs/audit/impl-records/phase-N/issue-NNNN-*.md` |
| Audit report | `docs/audit/<date>-<topic>.md` |
| Planning checklist | `docs/planning/<cascade>-checklist.md` |
| Translate | Regenerate `docs/zh-CN/<mirror>.md` via translation workflow |

## Data classification

| Path | Class | Handling |
|---|---|---|
| `docs/adr/**`, `docs/specs/**`, `docs/architecture/**` | public | Tier-2 approval to edit |
| `docs/audit/**` (non-log) | internal | Free edit for own-session reports |
| `docs/audit/overrides.log`, `governance-changes.log`, `commit-log.jsonl` | internal | Append-only |
| `docs/facts/generated.yaml` | generated-code | Auto-regenerated; do not hand-edit |
| `docs/identity/humans.yml` | user-data | CODEOWNERS-gated |
| `docs/planning/**` | internal | Free edit for cascade tracking |
| `docs/zh-CN/**` | public | Mirror of English source; never source-of-truth |

## Assessment rubric

In addition to root R1–R11:

| ID | Criterion | Verify with |
|---|---|---|
| R1-doc | ADR/spec edits cite tier-2 approval comment URL | Visual review |
| R2-doc | `docs/facts/generated.yaml` not hand-edited | `git diff docs/facts/generated.yaml` shows no manual hunks |
| R3-doc | If a public Python surface changed, ADR `governs.files` updated | `python -m scieasy.qa.audit.doc_drift` |
| R4-doc | English doc + zh-CN mirror updated together | `git diff --stat docs/zh-CN/` |
| R5-doc | Audit logs never deleted or rewritten | `git log --diff-filter=D docs/audit/*.log` empty |
