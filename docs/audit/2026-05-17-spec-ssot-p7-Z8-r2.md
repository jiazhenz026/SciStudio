# Phase 7 SSOT Audit — Agent #8 (Round 2)

Date: 2026-05-17  
Auditor: Z8  
Scope: phase6 SSOT (`docs/specs/INTERFACE_SPEC.md`) vs phase5 decision draft (`docs/planning/spec-ssot-draft-v3.md`) vs real code implementation.

## Verdict

存在 **P1 级不一致**，当前 SSOT 仍不能视为“已与实现一致”。

## Findings

### P1-1 `data-types.DataObject` 标记为 a 类但源码路径与签名均错误

- SSOT：`data-types.DataObject` 标记 `Status: a`，Source 指向 `src/scieasy/core/types/dataobject.py:L1-L1`。
- 代码：该文件不存在；`DataObject` 实际在 `src/scieasy/core/types/base.py`，且 `save()` 返回 `StorageReference`，并非 `None`。
- 结论：属于“标记为一致但实际不一致”。

### P1-2 `storage-backends.BackendRouter` 存在假接口

- SSOT：声明 `BackendRouter.dispatch(self, obj_or_type) -> StorageBackend`。
- 代码：`src/scieasy/core/storage/backend_router.py` 未定义 `dispatch`，实际是 `resolve/backend_for/backend_name_for/extension_for`。
- 结论：SSOT 存在“接口幻觉（假接口）”；a/b/c 分类应重判。

### P1-3 `execution-engine.EngineEvent` 字段缺失

- SSOT：`EngineEvent` 仅有 `event_type/block_id/data`。
- 代码：`src/scieasy/engine/events.py` 还包含 `timestamp: datetime = field(default_factory=datetime.now)`。
- 结论：a 类接口签名不一致，且属于关键事件模型字段丢失。

### P2-1 phase5 草案到 phase6 SSOT 迁移不完整（路径修复未收敛）

- phase5 草案（`spec-ssot-draft-v3.md`）已多次提醒存在 `L1-L1` 占位路径与伪路径风险。
- phase6 SSOT 仍保留多处 `L1-L1`、`L?`、不存在文件路径，说明决议草案的“清洗动作”未完整落地。

### P2-2 完整性风险：文档未稳定覆盖真实 API 面

- 在 REST/API 与 engine 相关条目中，部分接口虽然列出，但参数/返回模型与源码不完全对齐（以上 P1 已给出实例）。
- 这意味着“文档覆盖 ≠ 可执行契约覆盖”，应视为完整性缺陷，而非纯文字问题。

## 结论与处置建议

1. 先对所有 `Status: a` 进行自动化再验收（至少校验：文件存在、符号存在、签名一致）。
2. 对 `L1-L1/L?/not implemented` 条目批量回收，强制改为 c/d 并挂 issue。
3. 以代码抽取结果反向重写 `DataObject` / `BackendRouter` / `EngineEvent` 三处条目后再进入 Phase 8。
