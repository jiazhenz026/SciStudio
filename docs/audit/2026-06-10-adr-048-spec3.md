# ADR-048 SPEC 3 Developer-Docs Rewrite — Audit Report

## 1. Scope and method

This is an independent, with-context audit of ADR-048 SPEC 3 — the
delete-and-rewrite of the active package/block developer documentation — against
its governing contract
[`docs/specs/adr-048-developer-docs-refresh.md`](../specs/adr-048-developer-docs-refresh.md)
(FR-001..FR-031, SC-001..SC-011, the Documentation Inventory, and the recent-ADR
review tables). PR #1581 `[DO NOT MERGE]` is stacked on the SPEC 2 branch
`track/adr-048-spec2-plot-tools`; the audit diff base is that branch tip.

Method: read each rewritten page against the authoritative implementation source
(`src/scistudio/blocks/base/ports.py`, `registry/_scan.py`, the previewer specs),
verified the impact matrix against the spec's ADR review table, ran the required
test commands, reconciled the Codex review, and checked protected-file and
historical-doc integrity.

Owner constraint observed: **no scope reduction**. Every FR/SC in scope for this
slice was checked; nothing was waved through as "v1 / later".

## 2. Verdict

**APPROVE for owner-merge into the SPEC 3 umbrella.** No P1 findings. The
delete-and-rewrite is faithful to current contracts and to the recent-ADR
review; the new previewer/plot guide is complete; the stale-phrase guardrail
test is real and enforces SC-001. Two P2 and two P3 items below are
documentation-quality notes; none block merge.

### No stale `produced_type` (FR-006, SC-001) — VERIFIED

`OutputPort` (`src/scistudio/blocks/base/ports.py:33-34`) has **no**
`produced_type` field; it inherits only `accepted_types` from `Port`. Confirmed
no `produced_type=` *usage* anywhere in the diff. The three residual textual
mentions are all legitimate "do-not-write-this" documentation, not active
guidance:

- `block-contract.md:130-133` — explicit warning that the keyword is invalid and raises `TypeError`.
- `testing.md:363` — "`accepted_types`, never a nonexistent `produced_type`".
- `impact-matrix.md:74` — §3 "Stale patterns removed by this rewrite".

The scaffold `blocks.py.tpl` now emits `accepted_types=[Array]` (was the
`produced_type=None` bug). The guardrail test keys on the invalid usage form
`produced_type\s*=` (regex), so it permits the warnings but would catch any real
regression. SC-001 met.

**P2-1 — cli-integration plot-tool inventory is by-reference, not enumerated in the SPEC-3 diff.**
`docs/cli-integration.md` lists `scistudio-write-plot/SKILL.md` (line 131) but
the six `category:plot` MCP tools are enumerated in the skill + the new guide,
not re-listed in `cli-integration.md`. This satisfies FR-024 at the stack level
(SPEC 2 owns that file) and is acceptable for a stacked PR. Flagging only so the
owner is aware the literal tool names live in the skill/guide, not in
`cli-integration.md` itself. Not blocking.

**P2-2 — `previewers-and-plots.md` cites FR codes from a companion spec, not this one.**
Inline references like "(FR-028)", "(FR-009 / FR-010)", "(FR-022 / FR-024)",
"(FR-005)" in the guide map to the previewer/plot-tools companion specs, not to
`adr-048-developer-docs-refresh`'s FR numbering. A reader cross-checking against
the docs-refresh spec will not find those numbers. Cosmetic/traceability nit;
prose is otherwise accurate. Not blocking.

**P3-1 — `architecture-for-block-devs.md` "Resource hints (ADR-022)" appears in `block-contract.md` as `requires_gpu`/`cpu_cores` ClassVars marked "not on Block ABC".**
Correct and honest, but two pages (block-contract + architecture) both narrate
resource hints; minor duplication. Not blocking.

**P3-2 — Impact-matrix `last_updated: 2026-06-10` while CI clock has advanced.**

blocking.

## 6. Codex reconciliation (PR #1581)

One Codex review (`chatgpt-codex-connector`, commit `f061ca9f`) with two P2
inline comments. **Both target `docs/planning/adr-048-dispatch-prompts/s3-docs.md`
(the dispatch prompt), not the delivered documentation:**

1. **"Include the SDK integration test in required checks" (P2, s3-docs.md:60).**
   Reconciliation: **valid against the prompt, not the deliverable.** The spec
   lists `tests/integration/test_block_sdk_e2e.py` as a SPEC-3 test, and the
   dispatch prompt's required-check command omitted it. The delivered docs do
   not regress anything; this is advice for the docs agent's CI command. Since
   the docs changes are markdown/template-only and `test_new_block_package.py` +
   `test_blocks_template.py` cover the executable surface, the SDK e2e omission
   in the *prompt* is low-risk. Recommend the manager note it for the
   finalize-time check set; **not a blocker for the docs themselves.**

2. **"Don't mask required ruff failures" (P2, s3-docs.md:61).**
   Reconciliation: **valid against the prompt, not the deliverable.** The
   prompt's example used `ruff ... 2>&1 || true`, which would hide a real lint
   failure. The actual CI `Lint & Format` check on #1581 **passes** (8s) and is
   authoritative, so no masked failure shipped. The note is a correct critique
   of the prompt's illustrative command; the real gate is green. **Not a
   blocker.**

No Codex comment targets the rewritten docs, the impact matrix, the guide, the
scaffold template, or the guardrail test. Both are prompt-hygiene notes for the
manager to optionally fold into the finalize check set.
