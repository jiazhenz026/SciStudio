### `BlockState` — 8-state lifecycle enum for all block instances
- **Kind**: enum
- **Source**: `src/scieasy/blocks/base/state.py:L8-L18`
- **Public?**: yes
- **Inherits / implements**: `enum.Enum`
- **Signature**:
  ```python
  class BlockState(Enum):
      IDLE      = "idle"
      READY     = "ready"
      RUNNING   = "running"
      PAUSED    = "paused"
      DONE      = "done"
      ERROR     = "error"
      CANCELLED = "cancelled"   # ADR-018: user explicitly terminated
      SKIPPED   = "skipped"     # ADR-018: required upstream inputs missing
  ```
- **ClassVars / attributes**: n/a (enum members only)
- **Raises**: n/a
- **Notes**: `CANCELLED` and `SKIPPED` are semantically distinct — the former is user-initiated termination, the latter is automatic skip when upstream inputs are unavailable (both added by ADR-018). `BatchMode`, `InputDelivery`, and `BatchErrorStrategy` enums were removed by ADR-020.

**Valid state transitions** (from `_VALID_TRANSITIONS` in `block.py:L22-L31`):

| From | To (allowed) |
|------|-------------|
| IDLE | READY, SKIPPED, ERROR |
| READY | RUNNING, SKIPPED, ERROR |
| RUNNING | DONE, PAUSED, ERROR, CANCELLED |
| PAUSED | RUNNING, ERROR, CANCELLED |
| DONE | IDLE |
| ERROR | IDLE |
| CANCELLED | IDLE |
| SKIPPED | IDLE |

  ```python
  class Block(ABC):
      # ClassVars
      name: ClassVar[str] = "Unnamed Block"
      description: ClassVar[str] = ""
      version: ClassVar[str] = "0.1.0"
      subcategory: ClassVar[str] = ""

      key_dependencies: ClassVar[list[str]] = []
      config_schema: ClassVar[dict[str, Any]] = {"type": "object", "properties": {}}

      def __init__(self, config: dict[str, Any] | None = None) -> None: ...
      def transition(self, target: BlockState) -> None: ...
      def get_effective_input_ports(self) -> list[InputPort]: ...
      def get_effective_output_ports(self) -> list[OutputPort]: ...
      def validate(self, inputs: dict[str, Any]) -> bool: ...
      @abstractmethod
      def run(self, inputs: dict[str, Collection], config: BlockConfig) -> dict[str, Collection]: ...
      def postprocess(self, outputs: dict[str, Collection]) -> dict[str, Collection]: ...
      def process_item(self, item: Any, config: BlockConfig) -> Any: ...
      @staticmethod def pack(items: list[Any], item_type: type | None = None) -> Any: ...
      @staticmethod def unpack(collection: Any) -> list[Any]: ...
      @staticmethod def unpack_single(collection: Any) -> Any: ...
      @staticmethod def map_items(func: Any, collection: Any) -> Any: ...

      def persist_table(self, table: Any, output_dir: str | None = None) -> StorageReference: ...
      @staticmethod def _auto_flush(obj: Any) -> Any: ...
  ```
- **ClassVars / attributes**:
  - `name`, `description`, `version` — palette/UI metadata
  - `subcategory` — palette display subcategory (issue #588); base_category inferred from class hierarchy
  - `input_ports`, `output_ports` — static ClassVar port lists; overridden per-instance for variadic blocks
  - `variadic_inputs`, `variadic_outputs` — when `True`, effective ports are read from `self.config["input_ports"]` / `self.config["output_ports"]` (ADR-029 D8/D11)
  - `allowed_input_types`, `allowed_output_types` — constrain type dropdown in port editor UI (empty = accept anything)
  - `min_input_ports`, `max_input_ports`, `min_output_ports`, `max_output_ports` — optional count bounds for variadic ports (ADR-029 Addendum 1)
  - `dynamic_ports` — declarative descriptor `{"source_config_key": str, "output_port_mapping": {port_name: {enum_value: [TypeName, ...]}}}` for config-driven port type switching (ADR-028 Addendum 1 D1); validated at registry scan time
  - `execution_mode` — `ExecutionMode.AUTO` by default
  - `terminate_grace_sec` — 5.0s default grace period for SIGTERM before SIGKILL (ADR-019)
  - `key_dependencies` — declared external package dependencies
  - `config_schema` — JSON Schema for the block's config UI
- **Raises**:
  - `transition()`: `RuntimeError` on invalid state transition
  - `validate()`: `ValueError` on missing required port, type mismatch, constraint failure, or variadic port count violation
  - `unpack_single()`: `ValueError` if Collection length != 1
  - `persist_array()`, `persist_table()`: `RuntimeError` if no output_dir configured
  - `_auto_flush()`: `RuntimeError` if storage write fails
- **Notes**:
  - `process_item()` in base `Block` raises `NotImplementedError`; the `ProcessBlock` subclass overrides it with a different signature (3-arg with `state`)
  - `pack()` calls `_auto_flush()` on every item (Tier 3 safety net)
  - `parallel_map()` uses `ProcessPoolExecutor`; the docstring warns about memory for large items
  - `_auto_flush()` is exempt for `Artifact` with `file_path` set (ADR-031 D5 path-only transport)
  - `_auto_flush()` recursively flushes `CompositeData` internal slots (#436)
  - `persist_array()` has a Windows MAX_PATH workaround (redirects to short temp dir when `len(store_path) > 200`)
  - `get_effective_input_ports()` / `get_effective_output_ports()` return a **copy** of the class-level list for non-variadic blocks
