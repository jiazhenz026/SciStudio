### `MCP-S-002` тАФ `MCPServer` class
- **Kind**: class (lifecycle wrapper)
- **Source**: `src/scieasy/ai/agent/mcp/server.py:L69-L356`
- **Public?**: yes (in `__all__`)
- **Signature**:
  ```python
  class MCPServer:
      socket_path: Path
      project_dir: Path
      def __init__(self, socket_path: Path, project_dir: Path) -> None: ...
      async def start(self) -> None: ...
      async def stop(self) -> None: ...
      @property
      def port(self) -> int | None: ...
      async def serve(self) -> None: ...
      async def dispatch(self, request: dict) -> dict | None: ...
      async def _handle_client(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter) -> None: ...
  ```
- **Notes**:
  - `start()` is idempotent. On POSIX: Unix domain socket at `socket_path`, removes stale socket on startup. On Windows: TCP loopback on ephemeral port; port written to `<socket_path>.port` sentinel file.
  - `stop()` closes the server, removes socket file (POSIX) or port file (Windows).
  - `port` property: returns bound TCP port on Windows, `None` on POSIX.
  - `serve()` = `start()` + `serve_forever()` + `stop()` on teardown; used by standalone bridge.
  - `dispatch()` handles: `initialize` (returns capabilities), `tools/list`, `tools/call`. Returns `None` for JSON-RPC notifications (no `id` field тАФ strict MCP 2026 compliance).
  - Recognized JSON-RPC error codes: `_PARSE_ERROR=-32700`, `_INVALID_REQUEST=-32600`, `_METHOD_NOT_FOUND=-32601`, `_INVALID_PARAMS=-32602`, `_INTERNAL_ERROR=-32603`.
  - `_serialise_result()`: coerces FastMCP `ToolResult` to JSON. Prefers `structured_content`, falls back to `content` (iterates text blocks, parses JSON), then raw.
  - `tools/list` response includes `_meta: {category, mutation}` derived from tool tags.
