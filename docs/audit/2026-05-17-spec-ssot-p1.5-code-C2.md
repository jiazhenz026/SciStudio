### `TypeSignature` — dataclass describing semantic type of a DataObject via MRO chain

- **Kind**: dataclass
- **Source**: `src/scieasy/core/types/base.py:L60-L138`
- **Public?**: yes
- **Inherits / implements**: `dataclass`
- **Signature**:
  ```python
  @dataclass
  class TypeSignature:
      type_chain: list[str]
      slot_schema: dict[str, str] | None = field(default=None)
      required_axes: frozenset[str] | None = field(default=None)
  ```
- **ClassVars / attributes**:
  - `type_chain: list[str]` — ordered list of class names from most general (`"DataObject"`) to most specific
  - `slot_schema: dict[str, str] | None` — optional mapping of slot names to type names (composites only)

  - `from_type(cls, data_type: type) -> TypeSignature` — classmethod; walks MRO (excluding `object`), filters to `DataObject` and subclasses, captures `expected_slots` and `required_axes` ClassVar
- **Raises**: nothing explicitly
- **Notes**: ADR-027 D1. `from_type()` captures `required_axes` only when non-empty frozenset; `Array` itself has empty frozenset → `required_axes=None`.

### `DataObject` — base class for all first-class data objects

- **Kind**: class (plain Python, not ABC or Protocol)
- **Source**: `src/scieasy/core/types/base.py:L141-L554`
- **Public?**: yes
- **Inherits / implements**: `object`
- **Signature**:
  ```python
  class DataObject:
      Meta: ClassVar[type[BaseModel] | None] = None

      def __init__(
          self,
          *,
          framework: FrameworkMeta | None = None,
          meta: BaseModel | None = None,
          user: dict[str, Any] | None = None,
          storage_ref: StorageReference | None = None,
          metadata: dict[str, Any] | None = None,  # deprecated
      ) -> None: ...
  ```
- **ClassVars / attributes**:
  - `Meta: ClassVar[type[BaseModel] | None] = None` — plugin subclasses override with their Pydantic Meta model
- **Instance attributes** (set in `__init__`):
  - `_framework: FrameworkMeta` — always set; defaults to `FrameworkMeta()`
  - `_meta: BaseModel | None` — typed domain metadata, None on base
  - `_user: dict[str, Any]` — free-form JSON-serialisable dict; copied on construction
  - `_storage_ref: StorageReference | None`
  - `_transient_data: Any` — declared slot (ADR-031 Add2); never serialised
- **Properties**:
  - `framework -> FrameworkMeta` (read-only)
  - `meta -> BaseModel | None` (read-only)
  - `user -> dict[str, Any]` (read-only)
  - `metadata -> dict[str, Any]` — **deprecated**, returns `_user`; emits `DeprecationWarning`; removed in Phase 11
  - `dtype_info -> TypeSignature` — calls `TypeSignature.from_type(type(self))`
  - `storage_ref -> StorageReference | None` (read + write setter)
  - `_data -> Any` — backward-compat bridge property → reads/writes `_transient_data`
  - `_arrow_table -> Any` — backward-compat bridge property → reads/writes `_transient_data`
- **Methods**:
  - `with_meta(self, **changes: Any) -> Self` — immutable update of `meta` slot; derives new `framework`; raises `ValueError` if `meta is None`; base implementation does NOT propagate extra subclass `__init__` params (documented TODO(T-006))
  - `to_memory(self) -> Any` — materialise from storage via backend; emits `ResourceWarning` above 2 GB; raises `ValueError` if no `storage_ref`
  - `slice(self, *args: Any) -> Any` — partial read; delegates to backend `slice()`; raises `ValueError` if no `storage_ref`
  - `iter_chunks(self, chunk_size: int) -> Iterator[Any]` — chunk iteration; raises `ValueError` if no `storage_ref`
  - `get_in_memory_data(self) -> Any` — for persistence/export; prefers `to_memory()` then falls back to `_transient_data`; raises `ValueError` if neither available
  - `save(self, path: str | Path) -> StorageReference` — persists using BackendRouter; no-op if `_storage_ref` already set
  - `_validate_user(user: dict[str, Any]) -> None` (static) — JSON-serialisability check; raises `TypeError` on failure
  - `_reconstruct_extra_kwargs(cls, metadata: dict[str, Any]) -> dict[str, Any]` (classmethod) — hook for worker reconstruction; base returns `{}`
  - `_serialise_extra_metadata(cls, obj: DataObject) -> dict[str, Any]` (classmethod) — hook for worker serialisation; base returns `{}`
- **Raises**:
  - `ValueError` — `metadata=` and `user=` passed together; `to_memory`/`slice`/`iter_chunks` without `storage_ref`; `with_meta` without typed `meta`; `get_in_memory_data` with no data
  - `TypeError` — non-JSON-serialisable `user` dict
- **Notes**: ADR-027 D5. Backward-compat shim (`metadata=` kwarg, `.metadata` property) removed in Phase 11. `with_meta()` base has TODO(T-006) for Array override. `_data`/`_arrow_table` bridges removed once all callers migrated.

- **Public?**: yes
- **Inherits / implements**: `DataObject`
- **Signature**:
  ```python
  class Array(DataObject):
      required_axes: ClassVar[frozenset[str]] = frozenset()
      allowed_axes: ClassVar[frozenset[str] | None] = None
      canonical_order: ClassVar[tuple[str, ...]] = ()

      ) -> None: ...
  ```
- **ClassVars**:
  - `required_axes: ClassVar[frozenset[str]] = frozenset()` — base is empty; subclasses override
  - `allowed_axes: ClassVar[frozenset[str] | None] = None` — None = no restriction
  - `canonical_order: ClassVar[tuple[str, ...]] = ()` — preferred axis ordering for display/reorder
- **Instance attributes**:
  - `axes: list[str]` — per-instance axis labels
  - `shape: tuple[int, ...] | None`
  - `dtype: Any`
  - `chunk_shape: tuple[int, ...] | None`
  - `_transient_data: Any` — set from `data=` kwarg if provided
- **Properties**:
  - `ndim -> int` — `len(self.axes)`
- **Methods**:
  - `_validate_axes(self) -> None` — checks no duplicates, required_axes present, allowed_axes respected; raises `ValueError`
  - `__array__(self, dtype=None, copy=None) -> Any` — NumPy array protocol; calls `to_memory()`
  - `sel(self, **kwargs: int | slice) -> Array` — named-axis selection (ADR-027 D4); integer index drops axis, slice keeps; returns plain `Array` (not `type(self)`); for zarr backend uses `ZarrBackend.slice()` for partial read; always persists result to zarr via `get_output_dir()` or `tempfile`; raises `ValueError` for unknown axes, non-int/slice index, missing `storage_ref`
  - `iter_over(self, axis: str) -> Iterator[Array]` — yields slices along one axis; raises `ValueError` for unknown axis or `shape=None`
  - `with_meta(self, **changes: Any) -> Self` — override of base; propagates `axes`, `shape`, `dtype`, `chunk_shape`; raises `ValueError` if `meta is None`
  - `to_memory(self) -> Any` — override; checks `_storage_ref` first, then `_transient_data` fallback; raises `ValueError` if neither
  - `_reconstruct_extra_kwargs(cls, metadata) -> dict` — extracts `axes`, `shape`, `dtype`, `chunk_shape` from wire metadata; converts JSON lists to tuples
  - `_serialise_extra_metadata(cls, obj) -> dict` — emits `axes`, `shape`, `dtype`, `chunk_shape`; normalises dtype via `np.dtype()`
- **Raises**:
  - `ValueError` — axis validation failures; `sel()` errors; `to_memory()` without data; `with_meta()` without meta
- **Notes**: ADR-027 D1/D4. 6D axis alphabet: `{t, z, c, lambda, y, x}`. `sel()` always constructs plain `Array` to avoid `required_axes` violation on reduced slices. `Series` backend is zarr per BackendRouter (`Series` → zarr).

### `ZarrBackend` — Zarr-based storage for chunked N-D arrays

- **Kind**: class
- **Source**: `src/scieasy/core/storage/zarr_backend.py:L17-L96`
- **Public?**: yes
- **Inherits / implements**: `StorageBackend` (structural/Protocol)
- **Signature**:
  ```python
  class ZarrBackend:
      def read(self, ref: StorageReference) -> Any: ...
      def write(self, data: Any, ref: StorageReference) -> StorageReference: ...
      def slice(self, ref: StorageReference, *args: Any) -> Any: ...
      def iter_chunks(self, ref: StorageReference, chunk_size: int) -> Iterator[Any]: ...
      def write_from_memory(self, data: Any, path: str) -> StorageReference: ...
      def get_metadata(self, ref: StorageReference) -> dict[str, Any]: ...
  ```
- **Method details**:
  - `read(ref)` — opens zarr array read-only; returns `np.asarray(arr)`
  - `write(data, ref)` — `np.asarray(data)` → write-to-temp-then-rename (atomic); copies `axes` from `ref.metadata` to zarr attrs if present; returns updated `StorageReference` with `shape`, `dtype` in metadata

  - `write_from_memory(data, path)` — convenience wrapper: constructs `StorageReference(backend="zarr", path=path)`, delegates to `write()`
  - `get_metadata(ref)` — returns `{shape, dtype, chunks, ndim}`; adds `axes` if present in zarr attrs
- **Notes**: Uses `zarr.open_array` (not `zarr.open_group`). Write atomicity via `tempfile.mkdtemp` + `Path.rename()`. `axes` metadata preserved in zarr attrs on write and returned in `get_metadata`. `iter_chunks` only iterates axis 0.

### `ArrowBackend` — Apache Arrow / Parquet storage for columnar data

- **Kind**: class
- **Source**: `src/scieasy/core/storage/arrow_backend.py:L14-L79`
- **Public?**: yes
- **Inherits / implements**: `StorageBackend` (structural)
- **Signature**:
  ```python
  class ArrowBackend:
      def read(self, ref: StorageReference) -> Any: ...
      def write(self, data: Any, ref: StorageReference) -> StorageReference: ...
      def write_from_memory(self, data: Any, path: str) -> StorageReference: ...
  ```

- **Method details**:

- **Notes**: No atomic write (unlike ZarrBackend and FilesystemBackend). `slice()` semantics are column-selection, not row-range. `Series` uses zarr backend per BackendRouter (Arrow used for DataFrame only).

### `CompositeStore` — directory-of-slots storage for CompositeData

- **Kind**: class
- **Source**: `src/scieasy/core/storage/composite_store.py:L13-L173`
- **Public?**: yes
- **Inherits / implements**: `StorageBackend` (structural)
- **Signature**:
  ```python
  class CompositeStore:
      _MANIFEST_NAME = "manifest.json"

      def read(self, ref: StorageReference) -> Any: ...
      def write(self, data: Any, ref: StorageReference) -> StorageReference: ...
      def write_from_memory(self, data: Any, path: str) -> StorageReference: ...
  ```

- **Method details**:

  - `write(data, ref)` — `data` must be `dict[str, tuple[backend_name, slot_data]]`; writes each slot to `<base>/<slot_name>/data.{zarr|parquet|txt|bin}`; writes `manifest.json` last; returns ref with `slot_names` metadata; raises `TypeError` for non-dict data

  - `iter_chunks(ref, chunk_size)` — yields `(slot_name, slot_data)` tuples (ignores `chunk_size`)

- **ClassVars**:
  - `_MANIFEST_NAME = "manifest.json"`
- **Manifest JSON shape**:
  ```json

      "<slot_name>": {
        "backend": "zarr"|"arrow"|"filesystem",

    }
  }
  ```
- **Raises**:

  - `ValueError` — `_get_backend_for()` unknown backend name
- **Notes**: Write is **NOT atomic** (documented `.. warning::` + `.. todo::` in docstring).
