## Implementation rationale

- **Locked format identifiers preserved.** The `LoadMzMLFiles` mapping
  values intentionally retain the spec-mandated case (`mzML`, `mzXML`,
  `raw`, `d`) because they are persisted on `MSRawFile.Meta.format` and
  consumed by `_mime_for`. ADR-028 §D8 places no constraint on the
  values; the constraint is from the locked metadata schema.
- **`.d` directory case.** The original module-level `_detect_format`
  had `if path.is_dir() and path.suffix.lower() == ".d": return "d"` as
  a special branch. The base-class helper does not check `is_dir`, but
  it does not need to — `Path("foo.d").suffix == ".d"` regardless of
  whether the path is a file or directory on disk. The lookup is purely
  string-based on the suffix, so the directory case is covered without
  the extra branch.
- **`"raw"` fallback for unknown extensions.** The legacy helper had
  `return "raw"` as the final fallthrough so that vendor-proprietary
  extensions (e.g. `.wiff`, `.lcd`) would still produce an `MSRawFile`
  with an Artifact path. The base helper returns `None` for undeclared
  extensions, so `load()` explicitly maps `None → "raw"`. This is
  preserved as the loader-level fallback rather than re-introduced into
  `supported_extensions` so that `_detect_format` can still report the
  honest "I do not recognize this extension" answer to downstream
  consumers (e.g. `BlockRegistry.find_loader` in #1077).
- **`.xls` alias for read.** pandas' `read_excel` transparently handles
  both `.xls` and `.xlsx`. Declaring `.xls → xlsx` lets users drop a
  legacy spreadsheet into the loader without code paths having to
  branch on the extension. The save side declares only `.xlsx` because
  pandas' `to_excel` writes only the modern format.
- **SaveTable enum derivation.** Sorting the set of values guarantees
  determinism across Python versions and is a one-liner — no separate
  helper function or class-level computed property is required. The
  test asserts both that the enum equals the derived value AND that
  every enum value is a value in the ClassVar mapping (defends against
  drift if someone adds an entry to the ClassVar but forgets to
  refresh the enum).
- **`_read_table` signature change.** Each of the three table loaders
  was refactored to take `file_format: str` instead of re-parsing
  `path.suffix`. The error message was preserved at the loader level
  (`f"Loader: unsupported file format: {path.suffix}"`) so user-facing
  diagnostics did not change. Internal callers were updated in lockstep.

## Deviations from the spec

None of substance.

The dispatch spec referenced `_detect_format` removal as the only
mzML-specific cleanup; I also simplified `_probe_header` because the
`format_hint or _detect_format(path)` line was the only remaining
reason `_probe_header` referenced the deleted module-level helper.
`_probe_header` is module-private (leading underscore) and only called
from inside this file, so the signature change is local.
