## Implementation rationale

- **Compound-first lookup.** `_detect_format` walks `Path.suffixes` from the longest combination (`"".join(suffixes[start:])`) down to the single trailing suffix. This guarantees a path ending in `.ome.tif` resolves to a registered `.ome.tif` entry even when a `.tif` entry also exists, per ADR-028 §D8.
- **Case-insensitive normalization on both sides.** The lookup builds a `{key.lower(): value}` view of `supported_extensions` and lowercases each path suffix before probing. This lets subclasses register either `".tif"` or `".TIF"` and have paths from either casing match. Lower-casing the keys at lookup time (rather than mutating the ClassVar) avoids surprising mutation of the declared mapping.
- **Empty default semantics.** The base-class `supported_extensions = {}` short-circuits `_detect_format` to return `None` immediately, so existing subclasses (which do not yet declare the ClassVar) keep their current behavior. Downstream PRs (#1074–#1076) populate the mapping per subclass.

## Tests added

In `tests/blocks/io/test_io_block_abc.py::TestDetectFormat`:

- `test_compound_suffix_resolves_to_compound_entry` — `foo.ome.tif` resolves to the compound entry (`ome_tiff`) when both `.ome.tif` and `.tif` are registered.
- `test_single_suffix_resolves_to_single_entry` — a plain `foo.tif` resolves to the single entry (`tiff`) even when a compound entry also exists.
- `test_case_insensitive_match` — `foo.TIF` matches a loader declaring `.tif`.
- `test_unknown_extension_returns_none` — a path with an undeclared suffix returns `None`.
- `test_empty_supported_extensions_always_returns_none` — the base-class default (empty mapping) returns `None` for any path, including paths with otherwise plausible suffixes.
