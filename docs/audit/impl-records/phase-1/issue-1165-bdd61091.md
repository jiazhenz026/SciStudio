# Implementation record: #1165

Phase 1E sub-PR 2 of the ADR-042/043/044 cascade — completes the
Phase 1E TC-1E.5 + TC-1E.6 work that #1162 (sub-PR 1) deferred. With
this merge, all six Phase 1E TCs ship together and the §3.5 recursive
self-validation workflow goes live.

## Files modified

- `src/scieasy/qa/governance/weakened_ci_check.py` — NEW (TC-1E.5,
  ADR-043 §6.4). Detects the 14 CI-weakening patterns enumerated in
  §6.4: deleted-test-file, removed-test-function (AST diff),
  lowered-coverage-threshold, lowered-mutation-threshold (numeric key
  in pyproject — full mutmut integration deferred to Phase 1F per a
  module-level `TODO(#1113)`), unjustified-skip-or-xfail (AST scan of
  new `@pytest.mark.skip` / `@pytest.mark.xfail` decorators —
  justified when `reason=` cites a `#NNN`), disabled-lint-rule (ruff
  `select` shrink + `ignore` expansion), disabled-typecheck-flag
  (mypy `strict=*` / `disallow_*` / `warn_*` true→false +
  pyright severity downgrade), disabled-precommit-hook (`hooks: - id`
  removal), removed-ci-job (`.github/workflows/*.yml` job shrink),
  increased-pytest-timeout (`timeout = N` numeric increase),
  expanded-exemption-paths (any `exclusions:` list growth in either
  `.governance-paths.yaml` or `docs/governance/exemptions.yaml`),
  expanded-noqa-usage (new `# noqa` lines without a `#NNN`
  trailer, detected via `git diff --unified=0` scan),
  reduced-skill-list (`docs/skills/required.yaml` entry removal),
  reduced-honeypot-count (`.governance-paths.yaml`
  `honeypot_canaries:` `- path:` count decrease). A
  `Loosening-Approved: @<handle>` trailer in the commit messages
  between base and head marks every finding as
  `has_loosening_approval=True` + `blocking=False`; Tier-2
  verification of the approver remains a CI-side concern handled by
  `mod_pr_check`. Exports
  `verify_no_weakening(base_ref, head_ref) -> list[WeakeningFinding]`
  matching the §4.7 audit-module signature.
- `src/scieasy/qa/governance/path_filter.py` — NEW (TC-1E.6 part A,
  ADR-043 §3.5 audit fix C2). Loads `.governance-paths.yaml`
  `governance_paths:` glob list at runtime, compares it against the
  `git diff --name-only base..head` output, and appends a
  `touched=true|false` line (plus a `matched<<EOF_GOV ... EOF_GOV`
  heredoc when matched is non-empty) to `$GITHUB_OUTPUT`. The
  exclusive purpose is to replace the hand-maintained
  `on.pull_request.paths:` filter that previously covered only ~12
  of ~30 governance paths. Includes a self-contained `**`-aware
  glob matcher (split on `/`, `**` matches zero-or-more components)
  to avoid importing `pathspec`. Exports
  `filter(paths_yaml, base, head, output) -> bool` matching the
  §4.7 signature.
- `src/scieasy/qa/governance/workflow_sync_check.py` — NEW (TC-1E.6
  part B, ADR-043 §3.5 audit fix C2). Static-parses
  `.github/workflows/governance-modification.yml` and rejects:
  (1) reintroduction of `on.pull_request.paths:` /
  `on.pull_request.paths-ignore:` (shadow hand-list drift mode),
  (2) absence of the `path_filter` invocation step,
  (3) absence of the recursive `workflow_sync_check` step itself.
  Emits one `SyncCheckFinding` per drift category. Exports
  `verify(repo_root) -> list[SyncCheckFinding]` matching the §4.7
  signature.
- `.github/workflows/governance-modification.yml` — NEW. The §3.5
  recursive self-validation workflow. Trigger is `pull_request: {}`
  (no hand-list); path-filter runs dynamically via `path_filter`;
  the subsequent steps (`workflow_sync_check`, full audit,
  contradiction audit, monotonic, honeypot, tracker, weakened-CI)
  all gate on `steps.gov_paths.outputs.touched == 'true'`. Steps
  that depend on Phase-1F/1G/1H tools (`full_audit`,
  `contradiction_audit`, `adr_implementation_check --pr-aware`) are
  shipped as `continue-on-error: true` placeholders with explicit
  in-file `TODO(#1113)` markers per CLAUDE.md §7.6.
- `scripts/audit/weakened_ci_check.py` — NEW. CLI shim delegating
  to the module `main`.
- `scripts/audit/path_filter.py` — NEW. CLI shim.
- `scripts/audit/workflow_sync_check.py` — NEW. CLI shim.
- `src/scieasy/qa/governance/__init__.py` — MODIFY. Added module
  descriptions for the three new modules.
- `tests/qa/test_governance_weakened_ci_check.py` — NEW. 34 tests
  covering every one of the 14 weakening patterns (positive +
  negative case where applicable), enum-cardinality coherence with
  §6.4, the `Loosening-Approved` trailer ritual, AST handling of
  class-based tests + async tests + syntax-error tolerance, the
  multi-line ruff config form, the pyright severity-downgrade form,
  full workflow-file deletion, and the three CLI exit-code paths.
- `tests/qa/test_governance_path_filter.py` — NEW. 21 tests
  covering the matcher (literal / `*` / `**` / `**`-at-root /
  leading-dot files / does-not-cross-slash), the
  `filter()` entrypoint (touched=true, touched=false, empty
  registry, malformed YAML), the CLI (default paths-yaml relative
  to repo, missing paths-yaml graceful exit), the OSError fallback
  to stdout when `$GITHUB_OUTPUT` is unwritable, and the
  `_git_diff_names` failure / FileNotFoundError fallbacks.
- `tests/qa/test_governance_workflow_sync_check.py` — NEW. 11 tests
  covering the clean-workflow path, the
  workflow-missing path, the four drift categories
  (shadow-paths, shadow-paths-ignore, missing-path-filter-step,
  missing-workflow-sync-step), CLI exit codes (0/1), the
  line-number annotation on a shadow-filter finding, and a
  real-repo sanity test that the workflow file ACTUALLY shipped
  in this PR passes `verify()` without findings.
