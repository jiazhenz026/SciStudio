## Implementation rationale

- **Circular import broken via `_common.py`.** ADR-042 §5.2 lines
  618-639 (audit fix C1) is followed verbatim: the 9 primitives that
  were duplicated across `frontmatter.py` and `maintainers.py` now live
  in `_common`; both files import from it, and neither imports the
  other at module-load time. `frontmatter.py` still references
  `AgentRuntime` from `maintainers` — but only as a typing annotation
  on `agent_editable_allowlist`, and under `from __future__ import
  annotations` so the import is type-checker-only at definition time.
- **Forward-ref resolution.** Once both modules have loaded, the
  package `__init__.py` calls `ADRFrontmatter.model_rebuild()` and
  `SpecFrontmatter.model_rebuild()` so pydantic v2 resolves the
  string-form `list[AgentRuntime]` annotation into the real enum.
  Manager-default per SUMMARY TC-1A.1. A dedicated test
  (`test_model_rebuild_resolved_forward_ref`) feeds the validator a
  raw string `"Claude"` and asserts coercion to `AgentRuntime.CLAUDE`
  succeeds — this is the canary if a future refactor breaks the
  forward-ref resolution.
- **Denormalised totals validator on `AuditReport`.** Manager-default
  per SUMMARY TC-1A.3. The flat `findings` list reachable via
  `runs[*].findings` is the source of truth; `total_findings`,
  `by_severity`, and `by_drift_class` are denormalisations that must
  agree. The validator catches arithmetic mistakes by report
  producers early. Empty buckets are treated as absent (so producers
  may emit `by_severity={}` for zero-finding reports).
- **`extra="forbid"` everywhere.** Every model carries
  `model_config = ConfigDict(extra="forbid")` per ADR-042 §5.1.
  Tests assert this for each model.
- **mypy --strict clean.** All public functions / validators carry
  explicit return types; the `__init__.py` re-exports the public
  surface explicitly via `__all__` to satisfy `mypy --strict` and
  `RUF022` sorting.

## Deviations from the spec

- **X1** (`_common` not listed in ADR-042 `governs.modules`):
  shipped as-is. Errata to amend `governs.modules` deferred to a later
  cycle; closure check (1B.4) is not yet operational so this defers
  cleanly. Recorded in PR body.
- **X3** (vestigial imports): `Literal` was unused in `maintainers.py`
  after the rewrite and was dropped. `Field` is genuinely used in
  every model; no further unused imports beyond that.
- **Q5.1** (`pydantic[email]`): the existing dependency was
  `pydantic>=2.0` (no `[email]` extra). Upgraded to
  `pydantic[email]>=2.0` in `pyproject.toml`. `email-validator` is now
  a transitive dependency.
- **`model_rebuild()` at package init**: invoked for both
  `ADRFrontmatter` and `SpecFrontmatter`. `SpecFrontmatter` does not
  actually need it today (its only forward refs are stringised
  primitive types from `_common`), but defensive parity with
  `ADRFrontmatter` was preferred — it costs nothing and protects
  against future schema additions that may reference
  `maintainers.AgentRuntime`.
