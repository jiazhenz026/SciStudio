## Implementation rationale

- **Authority split (¬ß3.3 P0.3 audit fix).** ``mod_guard.py`` and
  ``mod_pr_check.py`` are kept in separate modules with explicit
  authority-limit docstrings. ``mod_guard.py`` has zero network imports
  (no ``urllib`` / ``http`` / ``requests``) ‚Äî this is a deliberate
  invariant that the audit pass can grep for. The CI module is the only
  authoritative approval check; the local one is best-effort UX that
  fails fast when a trailer is obviously missing and prints the
  remediation block.
- **Shared path matcher.** ``_glob_match`` is implemented once in
  ``mod_guard.py`` and re-imported into ``mod_pr_check.py``. This keeps
  LOCAL and CI views of "is this a governance path?" byte-identical.
  The matcher handles the four ``.governance-paths.yaml`` glob shapes
  the seed registry uses: bare path (``CLAUDE.md``), prefix-then-stars
  (``docs/adr/**``), stars-then-tail (``**/AGENTS.md``), and middle-stars
  (``a/**/b``) ‚Äî exercised explicitly in the parametrised matrix test.
- **9-of-14 monotonic axes in v1.** The five deferred axes all depend
  on data not seeded until later Phase 1 waves (1F frontmatter required-
  fields, 1G ratchet baselines, 1H workflow-v2 doc-drift exemption
  rules). Their absence is the conservative direction ‚Äî the regime
  errs on the side of "passes" when it cannot detect, which is fine
  while the corresponding data is being seeded. A single TODO(#1113)
  cluster comment in the source documents the deferral with the
  followup link.
- **Honeypot SHA pinning deferred.** ``.governance-paths.yaml`` line 90
  explicitly notes "SHA seeding is owned by TC-1B.7 orchestrators".
  TC-1E.4 honours that boundary: marker-presence detection is enough to
  satisfy the ¬ß3.5 workflow integration and lets the SHA-pinning layer
  arrive cleanly in 1B.7 without retro-fitting.
- **CLI exit codes follow temp_review precedent.** All four tools
  return ``0`` on OK, ``1`` on findings, and (where applicable) ``2``
  on configuration errors ‚Äî matching the convention established by
  ``scripts/audit/temp_review.py``.
