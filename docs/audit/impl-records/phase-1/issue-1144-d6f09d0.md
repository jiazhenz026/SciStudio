## Implementation rationale

* **Severity split (ast_lint)** — ADR-043 §4.2 names the AST patterns as
  "errors", but §4.3 reviewer intent treats some as soft signals
  (snapshot rationale, magic numbers). The implementation maps five
  structural blockers to `error` and the rest to `warning`. Reviewer
  override knobs (per-line suppression directive) are TODO-tagged for
  Phase 3.
* **`Finding.rule_id` namespace** — `TQAST-<pattern-slug>` for AST
  findings, `TQTF-impl-before-test` for the test-first verifier. The
  prefix lets CI annotations and baseline files filter unambiguously.
* **Subprocess seam (test_first_check)** — every `gh`/`git` call goes
  through a single `_run_capture(args)` function. Unit tests patch this
  one seam; the verifier itself is deterministic + platform-portable.
* **Test-first heuristic** — pairs `tests/**/test_<stem>.py` adds with
  `src/**/<stem>.py` adds by stem-name match. ADR-043 §4.1 acknowledges
  this is "best-effort"; emitting only `warning` severity by default
  contains the false-positive risk. `Backfill-Test:` trailer exempts
  legitimate debt-cleanup commits per §4.3.2.

## Deviations from investigation spec

* **Tracking-branch sequencing** — dispatch claimed Phase 1A schemas
  were on `main`, but they actually live on `origin/track/adr-042/1a-schemas`
  (3 commits ahead of `origin/main`). This Sub-PR branches off
  `track/adr-042/1a-schemas` and targets `track/adr-042/1f-test-quality`,
  so merging will fast-forward the 1F tracking branch to include 1A
  schemas. This matches the cascade design ("each tracking branch
  accumulates its dependencies before the Phase 1.5 owner decision") and
  is documented in the PR description.
