## Codex findings addressed

| Severity | Detector | Bug | Fix |
|---|---|---|---|
| P1 | `test_first_check._build_test_impl_pairs` | tracked only the first new test file per stem; later test files for the same stem were silently skipped | switched to per-stem list of test entries; each new test file paired against the earliest matching impl independently |
| P1 | `ast_lint._detect_asserts_on_mock_call_only` + `_is_assertion_node` | mock-call-assert name set (collection) and the "other meaningful assertion" exclusion drifted apart; ``assert_any_call`` / ``assert_has_calls`` / ``assert_not_called`` were classified as "meaningful" and suppressed the finding | lifted ``_MOCK_CALL_ASSERT_NAMES`` to a module constant; used in both branches |

## Tests added

* `test_asserts_on_mock_call_count_only_fires_for_all_mock_variants` —
  parametrized over the three new variants.
* `test_check_test_file_skips_nested_helper_named_test` — nested helper
  inside a fixture factory is ignored.
* `test_check_test_file_skips_methods_on_non_test_class` — methods named
  `test_*` on `class Helper:` are ignored.
* `test_iter_pytest_collectable_tests_returns_test_class_methods` — unit
  test on the new collection helper.
* `test_build_test_impl_pairs_yields_one_entry_per_test_file` — multiple
  test files for the same stem each get checked.
