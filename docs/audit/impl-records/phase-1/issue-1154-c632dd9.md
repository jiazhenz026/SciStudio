### Audit tools

- `src/scieasy/qa/audit/codemod_lint.py` ‚Äî ADR-042 ¬ß20.3 entry point.
  Three public functions:
  * `parse(codemod_path: Path) -> dict[str, Any]` ‚Äî returns the
    parsed-but-not-validated dict per the ¬ß20.3 stub literal.
  * `parse_model(codemod_path: Path) -> CodemodMeta` ‚Äî sibling that
    pydantic-validates (Q1B.9.1 manager default).
  * `check(repo_root: Path | None = None) -> list[Finding]` ‚Äî walks
    `tools/codemods/adr-*.py` and emits:
    * `codemod-lint.missing-metadata` (ERROR) ‚Äî no module docstring.
    * `codemod-lint.malformed-field` (ERROR) ‚Äî docstring missing
      `ADR:` or `Description:` keys.
    * `codemod-lint.schema-violation` (ERROR) ‚Äî pydantic
      ValidationError (e.g. ADR out of `[1, 9999]`).
    * `codemod-lint.adr-ref-mismatch` (WARNING) ‚Äî filename's
      `adr-NNN-<slug>` prefix disagrees with metadata `ADR:` field.
    * `codemod-lint.filename-format` (WARNING) ‚Äî filename doesn't
      match `adr-NNN-<slug>.py`.
    * `codemod-lint.missing-tests` (WARNING) ‚Äî empty `Tests:` block.
    * `codemod-lint.broken-test-ref` (WARNING) ‚Äî declared test path
      missing on disk.

- `src/scieasy/qa/audit/auto_generated_lint.py` ‚Äî ADR-044 ¬ß11.5 +
  ¬ß10.3 entry point. `check()` walks `docs/**/*.{md,txt}`, parses
  YAML frontmatter, skips non-`generation: auto` files, computes
  SHA-256 of each AG file's full body, and compares against
  `docs/audit/baselines/auto-gen.json` (Q1B.10.1 content-hash
  strategy). Emits `auto-generated-lint.no-baseline` (INFO) for
  first-run + `auto-generated-lint.hand-edit` (ERROR) for mismatched
  hashes. Hash comparison covers the full byte content so frontmatter
  drift is detected too. Per Q1B.10.2 the `Regenerated-At:` trailer
  escape hatch is deferred to a follow-up.

- `src/scieasy/qa/audit/doc_length_lint.py` ‚Äî ADR-044 ¬ß4.3 entry
  point. `check()` walks `docs/**/*.md`, filters to the four covered
  categories (`contributing/`, `user/`, `prod-agent/`, `doc-guide/`),
  skips `generation: auto` files, and counts:
  * non-empty source lines (frontmatter excluded) ‚Äî error `>120`,
    warning `>100`;
  * words (frontmatter + fenced code stripped) ‚Äî error `>600`.

  `length_exception_reason` frontmatter downgrades errors to INFO
  (auto-expiry mechanism deferred per ADR-044 ¬ß4.5). Exposed
  constants `ERROR_LINE_CAP = 120`, `WARNING_LINE_CAP = 100`,
  `ERROR_WORD_CAP = 600`. Helper `count_metrics(body_text)` returns
  `(line_count, word_count)` for reuse by other tools.

- `src/scieasy/qa/audit/skill_pointer_sync.py` ‚Äî ADR-044 ¬ß11.4 entry
  point. `check()` validates every `.claude/skills/*/SKILL.md`:
  * Frontmatter present + parseable, otherwise
    `skill-pointer-sync.missing-frontmatter` (ERROR).
  * `kind` field present + valid (`procedural` / `tool-wrapping` /
    `bootstrap-meta`), otherwise `missing-kind` (WARNING).
  * Body ‚â§30 lines (frontmatter excluded), otherwise
    `body-too-long` (WARNING) per ADR-044 ¬ß11.2 / ¬ß11.4.
  * No 3+ consecutive numbered list items, otherwise
    `procedural-duplication` (WARNING).
  * Pointer target extracted from first backticked path/module
    mention in body; per-kind shape regex:
    * procedural ‚Üí `docs/contributing/workflows/[a-z0-9-]+\.md`
    * tool-wrapping ‚Üí reference doc OR `scieasy.qa.audit.*` module
    * bootstrap-meta ‚Üí `docs/doc-guide/[a-z0-9-]+\.md`

    Shape mismatch emits `pointer-shape-wrong` (ERROR); missing
    target emits `target-missing` (INFO when 1D-dependent dir is
    absent; WARNING for tool-wrapping module paths).
  * Reverse closure: if `docs/contributing/workflows/` exists,
    workflow docs unreferenced by any procedural skill emit
    `workflow-unreferenced` (ERROR). If it doesn't, a single INFO
    `workflows-dir-absent` finding is emitted (dormant marker).

  Public helper `detect_skill_kind(frontmatter)` extracted for reuse
  by `workflow_sync`.

- `src/scieasy/qa/audit/workflow_sync.py` ‚Äî ADR-044 ¬ß11.5 + ¬ß12.3
  entry point. `run()` does the bidirectional workflow ‚Üî procedural
  skill closure. When `docs/contributing/workflows/` is absent (1D
  not yet shipped), returns a single INFO `workflow-sync.dormant`
  finding. Active path:
  * For each procedural skill with `kind: procedural`, the body's
    pointer is extracted (re-using the same regex pattern as
    `skill_pointer_sync`). No pointer ‚Üí `no-target` (WARNING).
    Pointer target missing on disk ‚Üí `target-missing` (ERROR).
  * For each workflow doc, the set of skills pointing at it is
    computed. Empty set ‚Üí `workflow-unreferenced` (ERROR). Skills
    pointing at the doc but absent from the doc's
    `related_skills` frontmatter list ‚Üí `related-skills-mismatch`
    (WARNING).

### Migration: `kind:` on existing skills (Q1B.10.3)

All nine `.claude/skills/speckit-*/SKILL.md` files now declare
`kind: bootstrap-meta` after the `description:` block. Mechanical
edit ‚Äî frontmatter only; the body is unchanged. Speckit skills are
bootstrap-meta because they govern the spec-kit meta-process (not a
runtime workflow, not an audit tool).
