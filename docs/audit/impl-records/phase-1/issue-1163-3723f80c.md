## Implementation rationale

- **¬ß6.4 vs ¬ß3.4 deliberate double-coverage.** Some axes appear in both
  the ``weakened_ci_check`` 14-pattern list and the
  ``monotonic_check`` 14-axis list (coverage threshold, pytest timeout,
  pre-commit hooks, mypy strictness, ruff rule sets). The two tools are
  independent; both run on every governance-touching PR. ¬ß6.4 is the
  GitHub-doctrine front line; ¬ß3.4 is the longer-tail axis sweep. The
  two are aligned on the same ¬ß3.4.2 loosening ritual for approval ‚Äî
  satisfying it suppresses ``blocking=true`` in both tools.
- **AST diff vs textual diff for test-function removal.** The ¬ß6.4
  table specifies "AST diff: function present in main, absent in PR."
  Implemented as: ``set(top_level_test_functions(base_src)) -
  set(top_level_test_functions(head_src))``. Class-scoped
  ``def test_*`` methods are deferred (TODO(#1113)) because they need a
  rename-vs-move canonicalisation that adds risk for v1.
- **path_filter output file semantics.** The function takes the
  ``output`` Path and APPENDS ‚Äî GitHub Actions accumulates step outputs
  in ``$GITHUB_OUTPUT`` and overwriting would clobber outputs from
  earlier steps. There is a dedicated regression test
  ``test_appends_to_output_file_preserving_existing_lines`` covering
  exactly this invariant.
- **workflow_sync_check handles PyYAML's ``on:`` quirk.** YAML 1.1
  defines ``on``/``off`` as boolean literals. PyYAML's default loader
  with ``yaml.safe_load`` honours this, so the workflow file's bare
  ``on:`` key parses as Python ``True``, not the string ``"on"``. The
  check looks up *both* keys so it works under either YAML profile ‚Äî
  defended by the ``test_pyyaml_on_key_boolean_coercion`` test.
- **No dedicated contradiction_audit tool yet.** ADR-043 ¬ß3.5 step
  "Re-run contradiction audit on modified ADRs" requires a tool that
  isn't part of Phase 1E. The workflow step calls the Phase -0.5
  ``temp_review.py`` as a stand-in (trailer + AST shape rules ‚Äî not a
  true ADR contradiction re-run) and TODO(#1113) marks the step for
  replacement when Phase 1F/1G ships the real tool.
