## Implementation rationale

- **Single file, stdlib only.** Design §3.1 mandates one CLI entry
  point with no submodules. The script uses `ast` for QA001/QA002,
  regex over file contents for QA004, `subprocess` against `git` for
  QA003/QA005, and `subprocess` against the existing `ruff` / `mypy`
  / `lint-imports` binaries for piggyback checks. No new PyPI
  dependency.
- **Cross-platform subprocess wrapping.** Windows `text=True`
  defaults to the legacy ANSI codepage (`gbk` on zh-CN locales),
  which crashes on non-ASCII output from `ruff`/`git`. The internal
  `_run_subprocess` helper forces `encoding="utf-8"` +
  `errors="replace"` and guards against `None` stdout/stderr.
- **Skip-when-absent piggybacks.** Mypy --strict and lint-imports
  return `(0, "skipped: …")` when `src/scieasy/qa/` does not exist
  (the current Phase -0.5 state) so the gate stays green until 1A
  schemas land.
- **QA004 scoping.** Design §2.8 says the frontmatter check fires
  only on *new* files *in the PR diff*. The full-scan code path
  (no `--ci` / no `--changed-files-only`) therefore skips QA004
  entirely — this prevents the sanity check from flagging
  historical ADR files that predate the temp regime. Migration to
  full pydantic schema validation in TC-1B.2 will widen the scope.
- **QA005 word boundaries.** The three patterns use `\b…\b` to
  avoid matching `git add ./path` (which is `git add .` followed
  by a path) accidentally. The `\.` form requires either whitespace
  or end-of-line after the dot to trigger.
- **CHANGELOG ordering.** Per the workflow-gate sequence, the
  CHANGELOG entry is committed *after* the docs (this file) but
  *before* PR submission. The temp_review.py CLI itself does not
  inspect CHANGELOG.md.

## Deviations from the spec

None of substance. Two minor refinements:

1. The design doc §3.4 shows the CI step appended as a free-standing
   block under `lint:`. The implementation adds it as the *last*
   step in the existing `lint` job (immediately after `ruff format
   --check .`) so the decommission revert is a single contiguous
   block removal.
2. The design's example output for the trailing summary said
   `0.32s`; the actual format is `f"{elapsed:.2f}s"` (e.g.
   `0.06s`). Same shape, same precision — just no fixed value.

## Tests added

`tests/audit/test_temp_review.py` covers each design §1.1 check:

- QA001: 4 tests (missing → fires, present → passes, init-only-imports
  exempt, init with real code fires).
- QA002: 3 tests (missing → fires with class name, with docstring

  satisfies; `Co-Authored-By:` satisfies; off-track branch is exempt).

  passes; non-ADR `.md` ignored; unchanged ADR not in candidate set
  is skipped).
- QA005: 4 tests (`git add -A` blocks; `git add .` blocks; `git add *`
  blocks; clean message passes).
- Piggyback: 2 tests (mypy --strict skipped when qa/ absent;
  lint-imports skipped when qa/ absent).
- Pyproject sanity: 1 test asserts `--cov-fail-under=70` and
  `timeout = 60` are still pinned.
- Exit-code semantics: 2 tests (env-error path when git unavailable;
  clean exit on fresh repo).
- Repo sanity: 1 integration test runs the CLI against the current
  repo and asserts exit 0.

Total: 25 tests, all passing locally at `pytest --timeout=60 --no-cov`.
