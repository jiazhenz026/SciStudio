## Notes on Judgment Calls

### Rule: ARCHITECTURE.md PRIMARY
The authority hierarchy means ADR-only matches do NOT downgrade a to d. For example:
- `_validate_dynamic_ports` (#86): xcheck flagged it as "unclear if exists"; C3 confirms it DOES exist in `registry.py`. D1's ARCH text references it directly. Classified a.
- `build_dag` (#94): D1's ARCH §6.1 pseudocode uses it as `self.graph = build_dag(workflow)`, making it a documented interface. Classified a.
- `lint endpoint` (#153): D1's ARCH §8.2 table includes `POST /api/lint/python`. Classified a.

### ADR-040 agent-provisioning coverage
D1 documents ARCH §10.2's provisioning contract (the function signature, the 4 sub-steps, the hook concept). Individual hook script names ARE documented in ARCH §10.2 (the hook file list appears explicitly). I treated these as **a** where D1 names the concept and C7 provides the detail, as long as both agree on what the artifact IS.

### ViewProxy in LazyList (#64)
This is a genuine **b**: D1's ARCH says ViewProxy is eliminated (ADR-031), but C3 code shows `collection[i].view().to_memory()` — ViewProxy is still present. The code contradicts the documented decision.

### git_author.json (#135)
Classified **b** because D1 explicitly names the file with a path, and C5 explicitly says no such file exists and shows the identity is hardcoded. This is a concrete discrepancy, not a "code hasn't been written yet" gap.

### CompositeData.slot_types() (#40)
D1 shows `{k: type(v) for k,v in _slots.items()}` (actual slot value types); code returns `expected_slots` ClassVar copy (declared schema types). These return semantically different things depending on whether slots are populated, making this a **b**.

### spawn_block_process (#111)
The ARCH signature is so different from the code signature that this is clearly **b**. However both have the function—ARCH documents the OLD signature (pre-ADR-019 refactor), code has the new one. Still both-present-disagree.
