- **Primary source**: `docs/architecture/ARCHITECTURE.md §5.1`
- **Supplementary sources**: ADR-018 §2.1, ADR-018 Addendum 1
- **Kind hint**: enum
- **Signature**:
  ```python
  class BlockState(Enum):
      IDLE = "idle"
      READY = "ready"
      RUNNING = "running"
      PAUSED = "paused"       # waiting for user (interactive/external)
      DONE = "done"

    RUNNING   → { DONE, PAUSED, ERROR, CANCELLED }
    PAUSED    → { RUNNING, ERROR, CANCELLED }
    DONE      → { IDLE }
    ERROR     → { IDLE }
    CANCELLED → { IDLE }
    SKIPPED   → { IDLE }
    ```
  - DONE/ERROR/CANCELLED/SKIPPED are terminal for a single workflow execution; transition back to IDLE only on workflow reset.

### `Block` — abstract base class for all blocks
- **Primary source**: `docs/architecture/ARCHITECTURE.md §5.1`
- **Supplementary sources**: ADR-004, ADR-017, ADR-018, ADR-019, ADR-020, ADR-027, ADR-029
- **Kind hint**: ABC
- **Signature**:
  ```python
  class Block(ABC):
      # --- Class-level declarations (overridden by subclasses) ---
      name: str = "Unnamed Block"
      description: str = ""
      version: str = "0.1.0"
      input_ports: list[InputPort] = []
      output_ports: list[OutputPort] = []
      execution_mode: ExecutionMode = ExecutionMode.AUTO
      terminate_grace_sec: float = 5.0       # grace period before SIGKILL (ADR-019)

      # Variadic port fields (ADR-029):
      variadic_inputs: ClassVar[bool] = False
      variadic_outputs: ClassVar[bool] = False
      allowed_input_types: ClassVar[list[type]] = [DataObject]
      allowed_output_types: ClassVar[list[type]] = [DataObject]

      def __init__(self, config: dict = None):
          self.config = BlockConfig(config or {})
          self.state = BlockState.IDLE

      def validate(self, inputs: dict[str, Collection]) -> bool: ...

      @abstractmethod
      def run(self, inputs: dict[str, Collection], config: BlockConfig) -> dict[str, Collection]: ...

      def postprocess(self, outputs: dict[str, DataObject]) -> dict[str, DataObject]: ...

      # Collection utilities (ADR-020):
      @staticmethod
      def pack(items: list[DataObject]) -> Collection: ...
      @staticmethod
      def unpack(collection: Collection) -> list[DataObject]: ...
      @staticmethod
      def unpack_single(collection: Collection) -> DataObject: ...
      @staticmethod
      def map_items(func, collection: Collection) -> Collection: ...
      @staticmethod
      def parallel_map(func, collection: Collection, max_workers=4) -> Collection: ...
  ```
- **Fields / attributes / args**:
  - `name`, `description`, `version` — class-level string metadata
  - `input_ports`, `output_ports` — list of Port instances
  - `execution_mode` — ExecutionMode enum
  - `terminate_grace_sec: float = 5.0` — grace period for SIGTERM before SIGKILL (ADR-019); Linux/macOS only
  - `variadic_inputs`, `variadic_outputs` — enable per-instance port addition (ADR-029)
  - `allowed_input_types`, `allowed_output_types` — type dropdown constraints for variadic ports (ADR-029)
  - `config` — BlockConfig (Pydantic validated param container)
  - `state` — BlockState (mutable runtime state)
- **Notes / behavior**:
  - `run()` always executes inside an isolated subprocess, never in the engine process (ADR-017)
  - Input signature is `dict[str, Collection]`; output is `dict[str, Collection]` (ADR-020)
  - `validate()` is called by engine before `run()`; return False to abort
  - `postprocess()` is optional hook for cleanup/logging/output transformation
  - All subprocess creation goes through `spawn_block_process()` factory (ADR-019)
  - `variadic_inputs/outputs` flags are carried in BlockSpec; frontend renders `[+]`/`[-]` buttons only when True (ADR-029)
  - Per-instance variadic port lists stored in `self.config["input_ports"]` / `self.config["output_ports"]` as JSON (ADR-029 D1)
  - `get_effective_input_ports()` / `get_effective_output_ports()` override returns per-instance ports when variadic config present (ADR-029, ADR-028 Addendum 1)

### `ProcessBlock` — deterministic data transformation
- **Primary source**: `docs/architecture/ARCHITECTURE.md §5.3`
- **Supplementary sources**: ADR-004, ADR-020, ADR-027 D7
- **Kind hint**: ABC (subclass of Block)
- **Signature**:
  ```python
  class ProcessBlock(Block):
      def setup(self, config: BlockConfig) -> Any:
          """Called once per run() before iterating. Return value passed to process_item() as state."""
          return None

      def teardown(self, state: Any) -> None:
          """Called in finally block. Default: no-op."""
          pass

      def process_item(
          self,
          item: DataObject,
          config: BlockConfig,
          state: Any = None,
      ) -> DataObject:
          raise NotImplementedError

      def run(self, inputs, config):
          primary = next(iter(inputs.values()))
          state = self.setup(config)
          try:
              if isinstance(primary, Collection):
                  results = []
                  for item in primary:
                      result = self.process_item(item, config, state)
                      result = self._auto_flush(result)
                      results.append(result)
                  output_name = self.output_ports[0].name if self.output_ports else "output"
                  return {output_name: Collection(results, item_type=primary.item_type)}
              else:
                  result = self.process_item(primary, config, state)
                  output_name = self.output_ports[0].name if self.output_ports else "output"
                  return {output_name: result}
          finally:
              self.teardown(state)
  ```
- **Fields / attributes / args**: `algorithm: str` (class attribute per palette diagram)
- **Notes / behavior**:
  - `setup()` is called inside worker subprocess after `TypeRegistry.scan()` and input reconstruction
  - `setup()` receives only config, not inputs; data-driven init belongs in `process_item()` with lazy init+cache
  - `teardown()` runs even if `process_item()` raises; use for cleanup (torch.cuda.empty_cache, conn.close)
  - `state` defaults to None; existing 2-arg `process_item(self, item, config)` overrides continue to work (backward compatible)
  - Default `run()` iterates primary input, auto-flushes each result (Tier 1 memory safety, ADR-020 Addendum 5)

### `SubWorkflowBlock` — workflow-as-block
- **Primary source**: `docs/architecture/ARCHITECTURE.md §5.3`
- **Supplementary sources**: ADR-004
- **Kind hint**: ABC (subclass of Block)
- **Signature**:
  ```python
  class SubWorkflowBlock(Block):
      def __init__(self, config: dict):
          super().__init__(config)
          self.workflow_ref = config["workflow_ref"]   # path or ID of the child workflow
          self.input_mapping = config["input_mapping"]  # parent port → child IOBlock
          self.output_mapping = config["output_mapping"] # child IOBlock → parent port

      def run(self, inputs, config):
          child_workflow = WorkflowLoader.load(self.workflow_ref)
          for parent_port, child_io_id in self.input_mapping.items():
              child_workflow.inject_input(child_io_id, inputs[parent_port])
          child_engine = DAGScheduler(child_workflow)
          child_engine.execute()
          return {
              parent_port: child_engine.get_output(child_io_id)
              for parent_port, child_io_id in self.output_mapping.items()
          }
  ```
- **Fields / attributes / args**:
  - `workflow_ref: str | Path` — path or ID of child workflow
  - `input_mapping: dict` — parent port name → child IOBlock node id
  - `output_mapping: dict` — child IOBlock node id → parent port name
- **Notes / behavior**:
  - Encapsulates an entire workflow DAG as a single reusable palette item
  - Frontend shows a "drill-down" button; clicking opens child workflow in nested canvas view
  - Supports recursive composition (child can contain SubWorkflowBlocks)
  - Subprocess isolation still applies: child DAGScheduler runs in a subprocess (ADR-017 table)

## 3. data-types

### `DataObject` — abstract base for all data
- **Primary source**: `docs/architecture/ARCHITECTURE.md §4.1 + ADR-031 D1`
- **Supplementary sources**: ADR-001, ADR-007, ADR-027 D5, ADR-031
- **Kind hint**: ABC
- **Signature** (ARCHITECTURE.md §4.1 classDiagram):
  ```python
  class DataObject:
      +framework: FrameworkMeta
      +meta: BaseModel
      +user: dict
      +dtype_info: TypeSignature
      +storage_ref: StorageReference
      +to_memory() Any
      +slice(*args) Any
      +iter_chunks(chunk_size) Iterator
      +get_in_memory_data() Any
      +with_meta(**changes) Self
      +save(path) None
  ```
  ADR-031 D1.2 defines internal slot names with underscore prefix:
  ```python
  _framework: FrameworkMeta
  _meta: BaseModel | None
  _user: dict[str, Any]
  _storage_ref: StorageReference | None
  ```
- **Fields / attributes / args**:
  - `framework: FrameworkMeta` — immutable framework-managed fields (created_at, object_id, source, lineage_id, derived_from); block authors do NOT mutate
  - `meta: BaseModel` — typed Pydantic BaseModel declared per subtype (e.g. FluorImage.Meta); base DataObject.meta is empty BaseModel
  - `user: dict[str, Any]` — free-form dict; framework never interprets; round-trips as JSON
  - `dtype_info: TypeSignature` — used by port system for connection validation; encodes class MRO chain
  - `storage_ref: StorageReference | None` — pointer to backing store
  - `.metadata` property — backward-compat shim returning `self.user` with DeprecationWarning; removed after Phase 11
- **Notes / behavior**:
  - DataObject MUST NOT carry raw data payloads (`_data`, `_arrow_table` prohibited on cross-boundary instances — ADR-031 D3)
  - `with_meta(**changes)` returns new instance with updated meta (immutable update pattern)
  - `get_in_memory_data()` is alias for `to_memory()`; subclasses may override (Text returns `content`, Artifact reads from `file_path`)
  - `to_memory()` emits 2 GB size warning for large data
  - ViewProxy eliminated (ADR-031 D2); no `view()` method; all data access methods live directly on DataObject

---

### `Array` — N-dimensional numeric data
- **Primary source**: `docs/architecture/ARCHITECTURE.md §4.1 + §4.3 + ADR-031 D1.3`
- **Supplementary sources**: ADR-002, ADR-027 D1, ADR-027 D4
- **Kind hint**: dataclass/ABC (subclass of DataObject)
- **Signature** (ARCHITECTURE.md §4.1 classDiagram + §4.3 + ADR-031):
  ```python
  class Array(DataObject):
      +axes: list[str]           # instance-level named axes
      +shape: tuple
      +dtype: numpy.dtype
      +chunk_shape: tuple
      +required_axes: ClassVar frozenset    # ADR-027 D1
      +allowed_axes: ClassVar frozenset     # ADR-027 D1
      +canonical_order: ClassVar tuple      # ADR-027 D1
      +sel(**kwargs) Array       # partial read along named axes; reads from storage
      +iter_over(axis) Iterator  # yield slices along one axis; each slice has storage_ref set
      +__array__(dtype) np.ndarray  # NumPy protocol; calls to_memory()
  ```
- **Fields / attributes / args**:
  - `axes: list[str]` — instance-level named axes; validated against class-level schema in `__init__` via `_validate_axes()`
  - `shape: tuple | None` — data dimensions (constructor parameter, ADR-031 D1.3)
  - `dtype: str | None` — element type (constructor parameter)
  - `chunk_shape: tuple | None` — storage chunk hint (constructor parameter)
  - `required_axes: ClassVar[frozenset[str]]` — axes every instance must have; empty on base Array
  - `allowed_axes: ClassVar[frozenset[str] | None]` — axes this class accepts; None means any; None on base Array
  - `canonical_order: ClassVar[tuple[str, ...]]` — preferred ordering for reorder operations; empty on base Array
- **Notes / behavior**:
  - 6D axis alphabet: `t` (time), `z` (depth), `c` (discrete channel), `lambda` (continuous spectral), `y` (vertical spatial), `x` (horizontal spatial)
  - `c` and `lambda` deliberately distinct: `c` = discrete label set; `lambda` = continuous physical quantity
  - `sel()` and `iter_over()` read from storage (not `_data`); slice results persisted to temp zarr, returned with `storage_ref` set (ADR-031 D1.3)
  - `TypeSignature.from_type(cls)` also records `required_axes` for port compatibility checks (ARCHITECTURE.md §4.1)
  - Axis validation: instance `axes` must be superset of `required_axes` and (if specified) subset of `allowed_axes`, no duplicates
  - Domain subtypes (Image, FluorImage, SRSImage, etc.) NOT in core; live in plugin packages registered via `scieasy.types` (ADR-027 D2)

### `Series` — 1D indexed data

### `CompositeData` — named heterogeneous slots
- **Primary source**: `docs/architecture/ARCHITECTURE.md §4.1`
- **Supplementary sources**: ADR-001, ADR-031 D1.7
- **Kind hint**: dataclass (subclass of DataObject)
- **Signature** (ARCHITECTURE.md §4.1 verbatim):
  ```python
  class CompositeData(DataObject):
      """A named collection of heterogeneous DataObjects."""

      def __init__(self, slots: dict[str, DataObject] = None, **kwargs):
          super().__init__(**kwargs)
          self._slots = slots or {}

      def get(self, slot_name: str) -> DataObject:
          """Retrieve a named slot (e.g. anndata.get('X') → Array)."""
          return self._slots[slot_name]

      def set(self, slot_name: str, data: DataObject) -> None:
          """Set or replace a named slot."""
          self._slots[slot_name] = data

      def slot_types(self) -> dict[str, type]:
          """Return {slot_name: DataObject subclass} for port constraint checking."""
          return {k: type(v) for k, v in self._slots.items()}

      @property
      def slot_names(self) -> list[str]:
          return list(self._slots.keys())
  ```
- **Fields / attributes / args**: `slots: dict[str, DataObject]` (via `_slots`); `slot_names: list[str]` (property)
- **Notes / behavior**:
  - Plugin-provided composite subclasses declare `expected_slots: dict[str, type]` (e.g. AnnData, SpatialData)
  - No AnnData/SpatialData in core (plugin-provided via `scieasy.types` entry-point)
  - CompositeData can nest CompositeData (SpatialData.table → AnnData)
  - Backend: directory of slot backends; manifest JSON maps slot names to storage refs
  - Port constraint checking via `slot_types()` and `slot_names` property

## 5. collection-transport

### `Collection` — homogeneous ordered transport wrapper
- **Primary source**: `docs/architecture/ARCHITECTURE.md §6.2`
- **Supplementary sources**: ADR-020, ADR-020 Addendum
- **Kind hint**: dataclass (NOT a DataObject subclass)
- **Signature** (ARCHITECTURE.md §6.2):
  ```python
  class Collection:
      """Homogeneous ordered collection of DataObjects.

      NOT a DataObject subclass — it is a transport wrapper, not a data type.
      Its type identity for port matching is determined by item_type.
      A single item is Collection with length=1. There is no special case.
      """
      items: list[DataObject]
      item_type: type          # e.g. Image, DataFrame — determines port compatibility

      def __getitem__(self, index): ...
      def __iter__(self): ...
      def __len__(self): ...
      def storage_refs(self) -> list[StorageReference]: ...
  ```
- **Fields / attributes / args**:
  - `items: list[DataObject]` — homogeneous ordered items
  - `item_type: type` — determines port compatibility; checked against port's `accepted_types`
  - `length: int` — property (also `__len__`)
  - `storage_refs() -> list[StorageReference]` — for cross-process serialisation
- **Notes / behavior**:
  - Engine treats every Collection as opaque unit: one Collection in, one Collection out, one subprocess, one ProcessHandle
  - Engine never unpacks, iterates, or inspects Collection contents
  - Port compatibility: Collection is transparent; port system checks `item_type` against `accepted_types`
  - `Collection` is used for all data flowing between blocks: single item = `Collection` with `length=1`
  - Collection construction enforces homogeneity: raises `TypeError` if items are not all `item_type` instances
  - For lineage recording: Collection items are unrolled into individual `data_objects` rows with `position` field (ARCHITECTURE.md §4.4)

### `DAGScheduler` — event-driven DAG execution
- **Primary source**: `docs/architecture/ARCHITECTURE.md §6.1`
- **Supplementary sources**: ADR-018, ADR-018 Addendum 1
- **Kind hint**: class
- **Signature** (ARCHITECTURE.md §6.1 verbatim):
  ```python
  @dataclass
  class RunHandle:
      """Returned by BlockRunner.run() for lifecycle tracking."""
      run_id: str
      process_handle: ProcessHandle
      result: asyncio.Future[dict[str, Any]]   # resolves when subprocess completes

  class DAGScheduler:
      def __init__(self, workflow: WorkflowDefinition, event_bus: EventBus,
                   runner: BlockRunner, resource_manager: ResourceManager,
                   process_registry: ProcessRegistry):
          self.graph = build_dag(workflow)
          self.block_states: dict[str, BlockState] = {}
          self.skip_reasons: dict[str, str] = {}
          self._active_tasks: dict[str, asyncio.Task[None]] = {}   # ADR-018 Addendum 1
          self.event_bus = event_bus
          self.runner = runner
          self.resource_manager = resource_manager
          self.process_registry = process_registry
          # subscriptions:
          event_bus.subscribe(CANCEL_BLOCK_REQUEST, self._on_cancel_block)
          event_bus.subscribe(CANCEL_WORKFLOW_REQUEST, self._on_cancel_workflow)
          event_bus.subscribe(PROCESS_EXITED, self._on_process_exited)

      async def execute(self): ...
      async def _dispatch(self, block_id: str): ...   # synchronous prelude; creates task
      async def _run_and_finalize(self, block_id, block, inputs, config): ...  # task body
      async def _on_block_done(self, event: EngineEvent): ...
      async def _dispatch_newly_ready(self): ...
      async def _on_cancel_block(self, event: EngineEvent): ...
      def _check_completion(self) -> None: ...
      async def _cancel_active_tasks_on_shutdown(self) -> None: ...
      def _propagate_skipped(self, failed_block_id: str): ...
  ```
- **Notes / behavior**:
  - Event-driven; subscribed to CANCEL_BLOCK_REQUEST, CANCEL_WORKFLOW_REQUEST, PROCESS_EXITED
  - `_dispatch()` is a synchronous prelude that creates asyncio.Task for `_run_and_finalize`; does NOT await runner (ADR-018 Addendum 1 Decision B)
  - `_active_tasks: dict[str, asyncio.Task]` tracks concurrent block tasks (ADR-018 Addendum 1)
  - `_dispatch_newly_ready()` rescans for READY blocks after each block completion AND retries resource-blocked dispatches
  - `_check_completion()`: all states in terminal set AND `not self._active_tasks`
  - Independent DAG branches execute concurrently via asyncio tasks
  - `_cancel_active_tasks_on_shutdown()` in execute() finally block: terminates subprocesses, awaits tasks (no leaks)
  - `can_dispatch()` refused → block stays in READY; retried on next resource release (ADR-018 Addendum 1 Decision B)

### `ProcessMonitor` — background crash detection loop
- **Primary source**: `docs/architecture/ARCHITECTURE.md §6.5`
- **Supplementary sources**: ADR-019
- **Kind hint**: class
- **Signature** (ARCHITECTURE.md §6.5 verbatim):
  ```python
  class ProcessMonitor:
      async def run(self):
          while True:
              for handle in self.registry.active_handles():
                  if not await handle.is_alive():
                      exit_info = await handle.exit_info()
                      self.event_bus.emit(ProcessExitedEvent(
                          block_id=handle.block_id, exit_info=exit_info))
                      self.registry.deregister(handle.block_id)
              await asyncio.sleep(self.poll_interval_sec)
  ```
- **Notes / behavior**: polls at 1-second interval; detects crashes, OOM kills, user OS task-manager kills; emits PROCESS_EXITED event

  );

      object_id               TEXT PRIMARY KEY,         -- FrameworkMeta UUID

      mtime_at_write          TEXT,

      block_execution_id      TEXT NOT NULL REFERENCES block_executions(block_execution_id),
