### 8.10 — New finding: _connect() file-backed WAL PRAGMA gap [new]

C5 notes that in file-backed mode, `_connect()` applies `PRAGMA foreign_keys=ON` on every connection (correctly noted) but does NOT re-apply `PRAGMA journal_mode=WAL` — WAL is only set on `_apply_pragmas_and_schema()` calls, which happen at construction time and at `__init__`'s bootstrap. Subsequent file-backed `_connect()` yields use a fresh connection with only `foreign_keys=ON` (from `_connect()` at L208), not WAL. This is a minor behavioral note that the spec should document.

**Verdict**: The `_apply_pragmas_and_schema()` is called at construction on both in-memory and file-backed stores (via the `with self._connect()` bootstrap at L177-L178). For file-backed stores, WAL is set at construction; subsequent per-call connections inherit WAL mode from the file (SQLite WAL mode persists in the file header). This is actually correct behavior — WAL is file-persistent, FK is per-connection. The spec note in C5 is technically accurate but the code behavior is sound.

**A5 ruling**: No spec change needed. Add a clarifying behavior note to `LineageStore` spec entry: "WAL mode persists in the file header post-construction; FK constraint enforcement is re-applied per connection."

**Priority**: P3 (behavior note, no fix needed).

### 9.6 — GitBinary discovery order verification

`git_binary.py:L77-L129` implements `locate()`. A5 enumerated the actual discovery order:

1. `SCIEASY_GIT_BUNDLE_ROOT` env var (if set) → path as bundle root
2. `sys.frozen` (PyInstaller) → `sys.executable.parent` as bundle root; macOS also adds `parent.parent / "Resources"`
3. Developer checkout: walk up `__file__.parents` to find `<repo>/desktop/` directory
4. For each bundle root: tries `resources/git/bin/git[.exe]`, `resources/git/cmd/git[.exe]`, `resources/git/mingw64/bin/git[.exe]`
5. System fallback: `shutil.which("git")`
6. Raises `BundledGitMissing` if all fail

**C5 describes 4 paths**: SCIEASY_GIT_BUNDLE_ROOT → PyInstaller → developer checkout → shutil.which. C5 says "developer checkout: walk up from __file__" — confirmed at L101-L105.

**Draft v1 §9 matches**: "4 discovery paths: SCIEASY_GIT_BUNDLE_ROOT → PyInstaller → `<repo>/desktop/` walkup → `shutil.which("git")`" — this matches the code. The macOS `../Resources` add is a sub-path of discovery #2, not a separate discovery path, so the "4 paths" count is correct.

**Verdict**: d-document confirmed correct.
