## Methodology

I sampled across all 13 modules + the 2 cross-cutting C-class entries. Sampling
was biased toward `Status: a` (claimed aligned) entries — those are the highest
risk because the SSOT preamble asserts they were "lifted verbatim from both
sides (they agree)", which means an audit that trusts the status label would let
them through. I additionally targeted entries that Z1–Z7 reports surfaced as
candidates for systemic patterns (broken `Source:` paths, line-range drift,
phantom symbols, `:L1-L1` placeholder).

Tools used: `Read` (verbatim file reads), `Grep` (symbol existence verification
across the repo), `git ls-tree` (file-existence sanity check).

Total entries enumerated from SSOT: **195 H3-level interface entries** across 13
modules + 2 cross-cutting C-class. Distribution: a=86, b=37, c=15, d=57. I
sampled ~25 entries in depth and spot-checked another ~15.

### F-Z8-003 [P1] — `BackendRouter.dispatch()` is a phantom method (a-class signature)

**Category**: phantom interface
**Module**: storage-backends

**SSOT entry** (`INTERFACE_SPEC.md:L2565-L2567`):

```python
class BackendRouter:
    def dispatch(self, obj_or_type: DataObject | type) -> StorageBackend:

```

**Code reality** (`src/scieasy/core/storage/backend_router.py:L8-L41`, verbatim):

### F-Z8-004 [P1] — `data-types.DataObject.save()` docstring chains the F-Z8-003 phantom

**Category**: consistency-drift on Status:a
**Module**: data-types

**SSOT entry** (`INTERFACE_SPEC.md:L1936-L1938`):

```python
def save(self, path) -> None:
    """Save to a destination path via the appropriate storage
    backend (BackendRouter.dispatch(type(self)))."""
```

**Code reality** (`src/scieasy/core/types/base.py:L475-L492`, verbatim):

```python
def save(self, path: str | Path) -> StorageReference:
    """Persist the data object to *path* using the appropriate backend.

    Returns the :class:`StorageReference` pointing to the persisted data.
    Also sets ``self._storage_ref`` so subsequent calls are no-ops.
    """
    if self._storage_ref is not None:
        return self._storage_ref

    from scieasy.core.storage.backend_router import get_router

    path_str = str(path)
    backend_name, backend = get_router().resolve(type(self))
    data = self.get_in_memory_data()
    ref = StorageReference(backend=backend_name, path=path_str)
    result_ref = backend.write(data, ref)
    self._storage_ref = result_ref
    return result_ref
```

**Drift summary**:
1. Return type: SSOT says `-> None`; code returns `StorageReference`.
2. Mechanism: SSOT cites `BackendRouter.dispatch(type(self))` (phantom);
   code calls `get_router().resolve(type(self))`.
3. Behaviour: SSOT omits the cache short-circuit (`if self._storage_ref is
   not None: return self._storage_ref`) and the post-write storage-ref
   sticking.

**Recommended action**: P1 — rewrite signature + docstring in SSOT.

### F-Z8-010 [P1] — `block-abc.BlockConfig` docstring uses wrong Pydantic attribute name (a-class)

**Category**: consistency-drift on Status:a
**Module**: block-abc

**SSOT entry** (`INTERFACE_SPEC.md:L96-L103`):

```python
class BlockConfig(BaseModel):
    model_config = ConfigDict(extra="allow")
    params: dict[str, Any] = {}

    def get(self, key: str, default: Any = None) -> Any:
        """Lookup key in self.params first, then self.__pydantic_fields__,
        finally return default."""
```

**Code reality** (`src/scieasy/blocks/base/config.py:L21-L33`, verbatim):

```python
def get(self, key: str, default: Any = None) -> Any:
    """Return a parameter value by *key*, falling back to *default*.

    Checks ``params`` first, then Pydantic extra fields ...
    """
    if key in self.params:
        return self.params[key]
    extras = self.__pydantic_extra__ or {}
    if key in extras:
        return extras[key]
    return default
```

**Drift**: SSOT names `self.__pydantic_fields__`; code uses
`self.__pydantic_extra__`. These are two different Pydantic v2 internals —
`__pydantic_fields__` is a `ClassVar[dict[str, FieldInfo]]` describing
declared fields; `__pydantic_extra__` is the per-instance dict of extra
keys when `extra="allow"`. The SSOT version would not work — declared
fields would never contain runtime-injected keys.

**Recommended action**: P1 (technically P2 — it's wrong but the same shape)
— replace with `__pydantic_extra__` (or use the prose from real docstring).

**Category**: consistency / partial
**Module**: block-abc

### F-Z8-013 [P2] — `data-types.DataObject` "5-slot model" overstates constructor surface

**Category**: consistency-drift on Status:a
**Module**: data-types

**SSOT entry** (`INTERFACE_SPEC.md:L1894-L1912`):

```python
class DataObject:
    """5-slot model (ADR-027 D5):
      - _framework: FrameworkMeta
      - _meta: BaseModel
      - _user: dict[str, Any]
      - _storage_ref: StorageReference
      - _transient_data: Any
    """
    _framework: FrameworkMeta
    _meta: BaseModel
    _user: dict[str, Any]
    _storage_ref: StorageReference | None
    _transient_data: Any | None
```

**Code reality** (`base.py:L171-L217`):

```python
def __init__(
    self,
    *,
    framework: FrameworkMeta | None = None,
    meta: BaseModel | None = None,
    user: dict[str, Any] | None = None,
    storage_ref: StorageReference | None = None,
    # Legacy kwarg, deprecated in Phase 10, removed in Phase 11.
    metadata: dict[str, Any] | None = None,
) -> None:
    ...
    self._transient_data: Any = None
```

**Drift**: the constructor takes **4** named slots (framework, meta, user,
storage_ref) plus the deprecated `metadata=` shim. `_transient_data` is
initialised internally to `None` and is **not a constructor parameter**.
Calling it a "5-slot model" in the docstring is misleading — _transient_data
is a runtime-populated bridge per ADR-031 Addendum 2, not a constructor slot.

The deprecated `metadata=` shim also doesn't appear in the SSOT block,
which means the "Detailedness invariant" ("every field/parameter") is
violated — a major API surface (deprecated but live until Phase 11) is
hidden.

**Recommended action**: P2 — rewrite as "4 init slots + 1 runtime-managed
bridge"; add the `metadata=` deprecation shim.
