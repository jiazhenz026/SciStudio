### `engine.events.EventBus` — Publish/subscribe dispatcher
- **Kind**: class (not a Protocol)
- **Source**: `src/scieasy/engine/events.py:L81-L141`
- **Public?**: yes
- **Inherits / implements**: —
- **Signature**:
  ```python
  class EventBus:
      def __init__(self) -> None: ...
      async def emit(self, event: EngineEvent) -> None: ...
      def subscribe(self, event_type: str, callback: Callable[..., Any]) -> None: ...
      def unsubscribe(self, event_type: str, callback: Callable[..., Any]) -> None: ...
  ```
- **ClassVars / attributes**:
  - `_subscribers: dict[str, list[Callable[[EngineEvent], None]]]` (defaultdict)
- **Raises**: Exceptions in callbacks are caught and logged; they never propagate to callers.
- **Notes**:
  - `emit()` iterates subscribers in registration order. If the callback returns a coroutine, it is awaited. Error isolation per ADR-018: a failing callback does not block subsequent callbacks.
  - `subscribe()` accepts both sync and async callables.
  - `unsubscribe()` silently ignores unknown callbacks (uses `contextlib.suppress(ValueError)`).
  - Multi-subscriber semantics: all registered callbacks for a given event type are called on each `emit()`.

#### `DAGScheduler._run_and_finalize` — Long-running task body for subprocess blocks
- **Source**: `src/scieasy/engine/scheduler.py:L335-L481`
- **Signature**:
  ```python
  async def _run_and_finalize(
      self,
      node_id: str,
      block: Any,
      inputs: dict[str, Any],
      config: dict[str, Any],
  ) -> None: ...
  ```
- **Notes**:
  - Awaits `runner.run(block, inputs, config)`.
  - Exception branches:
    - `asyncio.CancelledError`: re-raises (state managed by caller).
    - `BlockTerminalStateReportedError`: honours worker-reported terminal state (CANCELLED/ERROR/SKIPPED). Cancel-race guard: if block already CANCELLED, preserves CANCELLED state without re-emitting.
    - `Exception`: transitions to ERROR, emits `BLOCK_ERROR`.
  - On success: strips `__scieasy_env__` sentinel from result dict → `env_payload`, stores result in `_block_outputs`, calls `_persist_output_metadata`, transitions to DONE, emits `BLOCK_DONE` with full ADR-038 §3.2 payload.
  - `finally`: pops `node_id` from `_active_tasks`, calls `_check_completion()`.

#### `DAGScheduler._run_interactive` — In-process execution for interactive blocks
- **Source**: `src/scieasy/engine/scheduler.py:L482-L618`
- **Signature**:
  ```python
  async def _run_interactive(
      self,
      node_id: str,
      block: Any,
      inputs: dict[str, Any],
      config: dict[str, Any],
  ) -> None: ...
  ```
- **Notes** (#591/#594): 6-step protocol:
  1. State → PAUSED, emit `BLOCK_PAUSED`.
  2. Call `block.prepare_prompt(inputs, BlockConfig(**config))` → `prompt_data`.
  3. Emit `INTERACTIVE_PROMPT` with `{"workflow_id", "block_type", **prompt_data}`.
  4. Create `asyncio.Future`, store in `_interactive_futures[node_id]`, await it.
  5. State → RUNNING, emit `BLOCK_RUNNING`. Merge response into config as `interactive_response`. Call `block.run(inputs, BlockConfig(**enriched_config))`.
  6. State → DONE, emit `BLOCK_DONE`.
  - Cancellation check before step 5.
  - `finally`: pops `_interactive_futures[node_id]`, pops `_active_tasks[node_id]`, calls `_check_completion()`.

#### `DAGScheduler._on_cancel_block` — CANCEL_BLOCK_REQUEST handler
- **Source**: `src/scieasy/engine/scheduler.py:L1052-L1118`
- **Signature**:
  ```python
  async def _on_cancel_block(self, event: EngineEvent) -> None: ...
  ```
- **Notes** (ADR-018 Addendum 1): Three-branch cancel:
  1. `ProcessHandle` present → call `handle.terminate()`.
  2. No handle, active task → `task.cancel()`.
  3. No handle, no task → state set only.

  - Emits `BLOCK_CANCELLED`, propagates skip downstream.

#### `DAGScheduler` internal helpers
- `_gather_inputs(node_id) -> dict[str, Any]` — assembles inputs from upstream `_block_outputs` per edge definitions.
- `_instantiate_block(node_id) -> Any` — uses `BlockRegistry.instantiate()` when available; falls back to raw NodeDef.
- `_check_readiness(node_id) -> bool` — all predecessors must be DONE.
- `_drain_active_tasks() -> None` — awaits all active tasks in snapshot loop.
- `_cancel_if_active(block_id) -> None` — terminates subprocess/cancels task for a RUNNING block before re-dispatch.
- `_reset_upstream(block_id, visited)` — recursive non-DONE predecessor reset.
- `_reset_downstream_skipped(block_id)` — BFS SKIPPED-block reset.
- `_persist_output_metadata(node_id, result, workflow_id)` — ADR-038 §5.2 DataObject upsert; no-op when `_lineage_recorder` is bound (recorder owns write).
- `_build_block_done_data(...)` — assembles full ADR-038 §3.2 BLOCK_DONE payload.

```json

  "outputs": { "<port_name>": <wire_format_value>, ... },
```

1. If `parsed["outputs"]` present: extract `outputs` dict.

**Error envelope** (exit code 1):
```json
{"error": "<traceback_string>"}
```

**Worker stdin payload** (JSON, UTF-8 encoded bytes):
```json
{
  "block_class": "<module>.<ClassName>",
```

### `LineageRecorder` wiring in the engine layer:

**Instantiation**: `LineageRecorder` is injected into `DAGScheduler.__init__` as the optional `lineage_recorder` parameter. The import type hint in `scheduler.py` imports from `scieasy.core.lineage.recorder` (via TYPE_CHECKING).

**Events subscribed to** (from comment matrix in `events.py:L78` and scheduler usage):
- `BLOCK_DONE` — recorder writes `block_executions` + `data_objects` + `block_io` rows.
- `BLOCK_ERROR` — recorder writes terminal `block_executions` row.
- `BLOCK_CANCELLED` — same.
- `BLOCK_SKIPPED` — same.

**Methods the scheduler calls on the recorder** (from `scheduler.py`):
- `self._lineage_recorder.record_start(node_id)` — called in `_dispatch()` when lineage recorder is bound.
- `self._lineage_recorder._store` — accessed by `_resolve_lineage_store()` to get the underlying `LineageStore` for `upsert_data_object()`.

**Ownership rule** (ADR-038 §5.2 / Codex P1 #931): When a recorder is bound, `_persist_output_metadata()` is a no-op — the recorder owns all `data_objects` writes from its `BLOCK_DONE` handler. The scheduler writes `data_objects` only when no recorder is bound (CLI / direct test runners).

The SQL schema of `LineageStore` and the `block_executions` / `data_objects` / `block_io` row shapes are owned by module `lineage-db` (another agent's scope).

5. **`BLOCK_READY` event constant** — defined in `events.py` but the scheduler never emits it (blocks go directly IDLE → READY as internal state, no event fired).

6. **`RunHandle` in `BlockRunner.run()` return type** — a NOTE comment in `runners/base.py` says future evolution may return `RunHandle`. Currently `run()` returns `dict[str, Any]`; `RunHandle` is defined in scheduler.py but not produced by any runner.

7. **`engine.lineage_recorder` shim** — scheduled for removal 2026-11-15. Any spec that imports `LineageRecorder` from `scieasy.engine.lineage_recorder` will break after that date.

8. **`pty_control` transport model** — the module comment explicitly states the worker→engine IPC uses HTTP loopback (not a dedicated IPC channel or extension of the stdin/stdout protocol). The engine must set `SCIEASY_ENGINE_API_URL` and `SCIEASY_ENGINE_IPC_TOKEN` env vars for PTY control to work in production. If specs describe these as part of the standard runner protocol, that is inaccurate.

9. **`ResourceManager.memory_high_watermark` default** — constructor default is `0.90` (90%), not `0.80` as the class docstring says. The docstring comment at L122 says `memory_high_watermark=0.80` but the parameter default at L141 is `0.90`. Code wins; docstring is stale.

10. **`spawn_block_process()` vs `LocalRunner.run()`** — `spawn_block_process()` uses `subprocess.Popen` (sync); `LocalRunner.run()` uses `asyncio.create_subprocess_exec` (async) and `register_async_process()`. These are two different code paths. Specs that describe a single factory for subprocess creation should note `spawn_block_process` as the legacy/sync path still available for external callers, while `LocalRunner` uses the async path.
