# Summary

本轮仅审计代码层接口（函数/类/字段约定），不做架构优劣判断，不读取文档。
覆盖范围：
- **M03 Execution Engine**：`src/scieasy/engine/**`
- **M04 AI Runtime Services**：`src/scieasy/ai/**` + `src/scieasy/blocks/ai/**` + AI 运行时相关 API 路由（`src/scieasy/api/routes/ai.py`, `src/scieasy/api/routes/ai_pty.py`）

输出目标：梳理当前“可见 contract surface”（接口、schema、命名/事件约定、关键字段）。

# Findings by module

## M03 Execution Engine

### 1) Graph / Scheduling 接口面
- `dag.py`
  - `DAG` 数据结构字段：`nodes`, `adjacency`, `reverse_adjacency`, `edges`, `edge_map`
  - 核心函数：`build_dag(workflow)`, `topological_sort(dag)`, `get_root_nodes(dag)`, `get_leaf_nodes(dag)`, `get_downstream_blocks(dag, node_id)`
  - 异常：`CycleError`
- `scheduler.py`
  - `DAGScheduler` 入口：`execute()`
  - 调度内部协议（从函数命名可见）：dispatch、readiness、skip-propagation、cancel-workflow/block、interactive-complete、process-exited 回调路径
  - `RunHandle` 字段：`run_id`, `process_handle`, `result`
  - 输入/输出 object id 提取辅助：`_collect_object_ids(payload)`, `_object_ids_for_value(value)`（端口级 object_id 列表映射）

### 2) EventBus + 事件约定
- `events.py`
  - 事件消息 schema：`EngineEvent(event_type, block_id=None, data={}, timestamp=...)`
  - 总线 API：`EventBus.emit(event)`, `subscribe(event_type, callback)`, `unsubscribe(event_type, callback)`
  - 事件常量（字符串协议面）包含：
    - block 生命周期：`BLOCK_READY/RUNNING/PAUSED/DONE/ERROR/CANCELLED/SKIPPED`
    - cancel 控制：`CANCEL_BLOCK_REQUEST`, `CANCEL_WORKFLOW_REQUEST`
    - 进程：`PROCESS_SPAWNED`, `PROCESS_EXITED`
    - 工作流：`WORKFLOW_STARTED`, `WORKFLOW_COMPLETED`, `WORKFLOW_CHANGED`
    - 交互：`INTERACTIVE_PROMPT`, `INTERACTIVE_COMPLETE`
    - git 通知：`GIT_HEAD_CHANGED`

### 3) Checkpoint / Resume schema
- `checkpoint.py`
  - Checkpoint 顶层模型：`WorkflowCheckpoint`
  - 管理器：`CheckpointManager`
  - 持久化 API：`save_checkpoint(checkpoint, path)`, `load_checkpoint(path)`
  - 中间结果序列化接口：`serialize_intermediate_refs(block_outputs)`, `deserialize_intermediate_refs(data)`
  - Collection wire 约定：`{"_collection": true, "items": [...], "item_type": "..."}`
  - StorageReference-like dict 约定字段：`backend`, `path`, `format`, `metadata`

### 4) Resource / Dispatch gate schema
- `resources.py`
  - 请求模型：`ResourceRequest`
    - 字段：`requires_gpu`, `gpu_memory_gb`, `cpu_cores`, `max_internal_workers`
    - 计算属性：`effective_cpu`
  - 快照模型：`ResourceSnapshot`
    - 字段：`available_gpu_slots`, `available_cpu_workers`, `system_memory_percent`
  - 管理器：`ResourceManager`
    - 主要接口：`can_dispatch(...)` + allocation/release 类行为（由命名可见）
    - 事件联动：订阅 terminal/process exited 做资源回收

### 5) Runner / Process boundary
- `runners/base.py`
  - `BlockRunner` Protocol（执行器抽象边界）
- `runners/local.py`
  - `LocalRunner`（本地执行实现）
- `runners/process_handle.py`
  - `ProcessExitInfo`, `ProcessHandle`, `ProcessRegistry`
  - worker 启动相关函数：`build_worker_payload`, `register_async_process`, `spawn_block_process`
- `runners/worker.py`
  - 子进程 contract：`reconstruct_inputs(payload)`, `serialise_outputs(outputs, output_dir)`, `main()`
- `runners/process_monitor.py`
  - `ProcessMonitor`
- `runners/terminal_state.py`
  - `BlockTerminalStateReportedError`

### 6) PTY 控制边界
- `pty_control.py`
  - 请求 schema：`PtyTabSpec`
  - in-process handler API：`set_in_process_handler`, `get_in_process_handler`

# Open risks

1. **字符串常量事件契约分散风险**：EventBus 的事件类型是字符串常量，跨 engine / api / ai 多处消费，重命名或拼写偏差较难在静态层面发现。  
2. **多层 schema 并存风险**：EngineEvent dict-data、checkpoint wire dict、MCP Pydantic model 并行存在，字段演进若不同步，可能出现“运行通过但语义漂移”。  
3. **PTY 映射状态一致性风险**：AI PTY tab_id ↔ block_run_id ↔ run_dir 映射是运行时状态，若进程异常退出/重连，存在悬挂映射或回放一致性问题。  
4. **Provider 能力探测口径风险**：`ai.py` 探测与 `blocks/ai/providers.py` 实际调用路径分离，可能出现“探测可用但调用失败”或反之。  
5. **Worker 边界序列化风险**：engine worker 的 reconstruct/serialise 与 AI block 侧输出约定并行演进时，若缺统一 version tag，可能出现隐式不兼容。
