- **A（对齐）**：文档契约与代码清单基本一致，且没有影响执行语义的一致性缺口。

## 2. 重点说明（仅 C 与高风险 B）

### M06 = C（显著偏差）

- 文档侧将 workflow graph 作为后端唯一事实源，并强调统一执行/校验语义。
- 代码审计清单显示 `validate_workflow(...)` 在 `registry is None` 时会跳过类型/端口等 registry 相关检查，保留结构级检查；registry 可用时才执行更强校验。
- 该差异会使**同一份工作流定义**在不同运行上下文得到不同深度诊断，属于“契约强度随环境变化”的显著偏差。

### B 级中优先收敛项

1. **M02/M03/M05：字符串协议面较多**（事件名、WS message type、部分 payload key）。建议后续收敛到结构化 schema（至少在边界层统一校验模型）。
2. **M01/M06：文档口径一致性**需补齐（核心类型命名与 workflow 校验强制等级说明），避免读者把“规范目标”误认为“当前强保证”。
3. **M07：前后端边界注释化依赖**应提升为显式协议文档，减少隐式耦合。
