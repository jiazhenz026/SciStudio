## 2. Scope

- In scope:
  - User-facing docstrings of public (`__all__` / `@stable` / `@provisional`,
    non-underscore) classes, functions, and blocks under the public modules:
    `core/types`, `core/storage`, `core/lineage`, `core/versioning`,
    `core/meta`, `blocks/base`, `blocks/process`, `blocks/io`, `blocks/app`,
    `blocks/subworkflow`, `blocks/code`, `blocks/ai`, `blocks/registry`,
    `plot`, `previewers`.
  - Public **attributes and properties** of those public classes
    (e.g. block `ui_icon`/`ui_color`, config `ClassVar`s): add attribute
    docstrings so they render in the generated reference with descriptions.
    Verified end-to-end (mkdocs `--strict`); no build-tooling change needed.
  - `docs/contributing/docstring-style.md` (the convention; manager-owned).
- Out of scope:
  - Internal-only modules' docstrings (tracked by umbrella `#1843`).
  - Any runtime behavior change. Docstrings/comments only.
  - `docs/user/reference/**` (generated by `scripts/docs/build_reference.py`).
- Protected paths:
  - `docs/user/reference/**` is generated — do not hand-edit.
- Deferred work (tracked under umbrella `#1843`):
  - Whole-repo internal-module docstring cleanup (future module-scoped PRs).
  - De-jargon user-facing **runtime strings** that still cite internal records
    (CodeBlock diagnostics' `suggested_target` "ADR-041"/"Track A" prose,
    `SubWorkflowBlock` authoring-only `RuntimeError` message "ADR-044"). Left as
    behavior-adjacent / test-coupled strings out of this docstring-only PR; the
    structured `diagnostic.reference` ADR field is a deliberate traceability
    field and is kept.
  - Publishing additional canonical reference roots (`scistudio.plot` — 40
    public symbols, `scistudio.previewers.helpers`) so their now-cleaned
    docstrings render in the generated reference. This changes the ADR-052
    published surface and needs ADR sign-off; their docstrings are already
    clean for deep-path readers and AI help text.
  - CodeBlock config doc-accuracy nits (fields documented as active that are
    reserved/ignored at run) and uneven `Example:` depth on read-only result
    dataclasses.
