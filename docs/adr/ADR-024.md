# ADR-024: Frontend Bundling, SPA Serving, And The scieasy gui Command

## 1. Decision Summary

SciEasy ships the built frontend with the Python package and exposes a
zero-configuration launch command:

```text
pip install scieasy
scieasy gui
```

FastAPI serves `/api/*` and `/ws` as backend routes. All other paths fall back
to the single-page application when built frontend assets are available.
Developers may still run the Vite dev server, but end users should not need
Node.js to launch the GUI.

### 1.1 Problems Addressed

| Problem | Risk | ADR-024 response | Detailed section |
|---|---|---|---|
| Non-code users cannot be asked to run Node.js and Vite | The GUI would be developer-only | Build the frontend into package assets and serve it from FastAPI | Section 3 |
| SPA deep links need fallback routing | URLs such as `/projects/123/workflows` would 404 | Mount `SPAStaticFiles` after backend routes | Section 4 |
| Users need one command to open the GUI | They would have to know uvicorn and browser URLs | Add `scieasy gui` and keep `scieasy serve` for headless API use | Section 5 |
| API and WebSocket routes must not be shadowed | A catch-all SPA mount could break backend calls | Reserve `/api/*` and `/ws`, mount static last, and test the boundary | Section 8 |

## 2. Context

ADR-023 makes the frontend a primary user surface. The target user includes
scientists who install Python packages but do not develop JavaScript apps. A
separate `npm install` plus `npm run dev` process is acceptable for frontend
development, not for normal use.

## 3. Frontend Bundle Decision

The React/Vite frontend is compiled before packaging and included in the Python
wheel as package data. The installed package may contain:

```text
scieasy/
  api/
    app.py
    static/
      index.html
      assets/
```

The repository also supports an editable-development fallback: if
`frontend/dist/index.html` exists in the source checkout, the API app may serve
that build before falling back to packaged static files. This keeps local
development practical without changing the end-user packaging contract.

Generated static assets are not hand-authored source. They should not be
committed as normal development files.

## 4. SPA Routing Boundary

Backend paths are reserved:

| Path | Owner |
|---|---|
| `/api/*` | FastAPI REST routes |
| `/ws` | FastAPI WebSocket endpoint |
| all other paths | SPA fallback when built assets exist |

The SPA mount is registered after all API and WebSocket routes. Unknown
non-API paths return `index.html`; real static assets return their file
contents. If no built frontend exists, root redirects to API docs rather than
returning a broken page.

## 5. CLI Decision

`scieasy serve` starts the API server for headless or developer workflows.

`scieasy gui` starts the same FastAPI app, sets the local engine API URL
environment needed by engine-spawned workers, and opens the user's browser
unless `--no-browser` is supplied.

The CLI command is the supported end-user launch path for the packaged GUI.

## 6. Scope

In scope:

- built SPA asset discovery and serving;
- FastAPI route ordering and fallback behavior;
- `scieasy gui` command;
- `scieasy serve` as the API-only server command;
- package/build configuration needed to include static assets.

Out of scope:

- Electron or native desktop shells;
- requiring Node.js at install or first launch;
- serving API routes outside the reserved `/api/*` prefix;
- committing generated SPA build output as source.

## 7. Build And Packaging Expectations

Release automation builds the frontend and copies the output into the package
static directory before wheel creation. Vite assets must use relative paths so
the SPA works from local FastAPI routes and deep-link fallback paths.

`pyproject.toml` remains the Python package authority. The console script
entry-point exposes `scieasy`, and the Typer app defines the `gui` and `serve`
commands.

## 8. Verification And Tooling Impact

Tests should verify:

- `SPAStaticFiles` returns `index.html` for deep SPA routes;
- real static assets are served directly;
- `/api/*` routes are not intercepted by the SPA mount;
- app creation redirects root to `/docs` when no SPA exists;
- app creation serves SPA index when static assets exist;
- CLI `gui` and `serve` call uvicorn with the FastAPI factory target and
  expected host/port behavior.

### 8.1 Signature-Level Contracts

```text
def create_app() -> FastAPI: ...


def _resolve_spa_static_dir() -> Path | None: ...


class SPAStaticFiles:
    def lookup_path(self, path: str) -> tuple[str, os.stat_result | None]: ...


def serve(host: str = "0.0.0.0", port: int = 8000) -> None: ...


def gui(
    port: int = typer.Option(8000, help="Port for the API server"),
    no_browser: bool = typer.Option(False, "--no-browser", help="Do not open browser automatically"),
) -> None: ...
```

## 9. Consequences

- End users can launch the GUI without a JavaScript toolchain.
- Frontend and backend versions are aligned by the same wheel.
- Developers still need frontend tooling only when changing the frontend.
- Route ordering becomes a correctness requirement and must stay covered by
  tests.

## 10. Alternatives Considered

Requiring Node.js at install time was rejected because it makes the end-user
path too fragile.

Electron was rejected because it adds a heavier distribution and update model.

Streamlit or Gradio was rejected because SciEasy needs a custom graph editor.

Shipping the frontend as a separate package was rejected because it creates
version synchronization and installation friction.
