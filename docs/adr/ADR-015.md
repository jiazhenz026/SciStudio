# ADR-015: Inclusive Strategy To Wrap Existing Tools

## 1. Decision Summary

SciEasy's default strategy is to wrap existing scripts, applications, and
pipelines rather than replace them.

### 1.1 Problems Addressed

| Problem | Risk | ADR-015 response | Detailed section |
|---|---|---|---|
| Researchers already depend on mature tools | Reimplementation would delay adoption and fragment workflows | Provide wrapping mechanisms | Section 3 |
| Different tools need different wrappers | One adapter style cannot cover scripts, apps, and pipelines | Use CodeBlock, AppBlock, and SubWorkflowBlock | Section 3 |

## 2. Decision

SciEasy integrates existing work through `CodeBlock` for scripts, `AppBlock`
for external software, and `SubWorkflowBlock` for reusable pipelines. Tighter
domain integrations may exist later, but replacement is not the default
architecture.

## 4. Alternatives Considered

Reimplementing common tools was rejected. Requiring formal adapters for every
tool was rejected because file exchange and scripts provide immediate value.
