# ADR-004: Five Block Categories Plus SubWorkflowBlock

## 1. Decision Summary

SciEasy keeps the block taxonomy small: IO, process, code, app, AI, and
subworkflow blocks. Categories describe execution pattern, not every possible
data operation.

### 1.1 Problems Addressed

| Problem | Risk | ADR-004 response | Detailed section |
|---|---|---|---|
| Users need a memorable block taxonomy | Too many block kinds make the palette hard to learn | Use five execution categories plus subworkflows | Section 3 |
| Runtime dispatch differs by execution pattern | A single mode flag would hide lifecycle differences | Keep distinct block base classes | Section 4 |

## 2. Context

SciEasy must cover pure transformations, user scripts, external apps, AI
steps, IO, and reusable nested workflows without forcing users through a deep
taxonomy.

## 3. Decision

The accepted categories are `IOBlock`, `ProcessBlock`, `CodeBlock`,
`AppBlock`, `AIBlock`, and `SubWorkflowBlock`. Merge, split, filter, and
similar operations are process blocks. Export is IO. CLI or GUI tools are app
blocks when they use an external process boundary.

## 4. Scope And Consequences

Separate base classes let the runtime reason about auto, interactive, and
external execution without guessing from arbitrary flags. The tradeoff is that
some cases are fuzzy: reusable typed Python logic should become a
`ProcessBlock`; one-off user scripts should use `CodeBlock`.

### 4.1 Signature-Level Contracts

```python
class IOBlock:
    ...

class ProcessBlock:
    ...

class CodeBlock:
    ...

class AppBlock:
    ...

class AIBlock:
    ...

class SubWorkflowBlock:
    ...

class ExecutionMode:
    ...
```

## 5. Verification

- `frontmatter_lint` accepts this ADR.
- Registry and block tests resolve the category classes.
- Signature drift resolves section 4.1 to implementation symbols.

## 6. Alternatives Considered

More categories were rejected as cognitive overhead. A single block class with
mode flags was rejected because app, code, process, AI, IO, and subworkflow
lifecycles have different runtime contracts.
