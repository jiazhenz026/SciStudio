# ADR-003: Broadcast As Explicit Utility, Not Implicit Type-System Behaviour

## 1. Decision Summary

SciStudio provides explicit named-axis broadcast helpers for block authors, but
the port system and DAG scheduler never broadcast data automatically.
Broadcasting is a domain decision: the block author must choose the target
axes and the function to apply.

### 1.1 Problems Addressed

| Problem | Risk | ADR-003 response | Detailed section |
|---|---|---|---|
| Low-dimensional data often applies across higher-dimensional arrays | Each block would reimplement axis iteration | Provide `broadcast_apply` as an opt-in utility | Section 3 |
| Shape compatibility is not semantic compatibility | Automatic broadcast could silently combine unregistered images or mismatched modalities | Keep broadcasting out of ports and scheduler | Section 4 |

## 2. Context

ADR-002 gives `Array` values named axes. That makes common patterns possible:
apply a `(y, x)` mask to every `mz` slice, apply a baseline to each spectrum
in a batch, or apply an ROI across time. NumPy-style automatic broadcasting is
not enough because matching shape does not prove co-registration, pixel size,
field of view, or modality compatibility.

## 3. Decision

SciStudio uses `scistudio.utils.broadcast.broadcast_apply` as the explicit helper
for this pattern. The helper validates named-axis alignment, iterates the
specified `over_axes`, and calls the user-supplied function for each slice.

Blocks remain responsible for deciding when broadcasting is valid. If a mask
must be registered before use, registration is a separate workflow step, not
an implicit framework guess.

## 4. Scope

### 4.1 In Scope

- explicit broadcast helper behavior;
- named-axis validation before broadcast;
- clear `BroadcastError` failures when axes do not align.

### 4.2 Out Of Scope

- automatic port-level or scheduler-level broadcasting;
- image registration or coordinate-system inference;
- storage-backed chunk iteration refinements governed by later ADRs.

## 5. Affected Artifacts

| Artifact | ADR-003 requirement |
|---|---|
| `src/scistudio/utils/broadcast.py` | Defines explicit broadcast utility and errors. |
| `tests/core/test_broadcast.py` | Verifies axis validation, iteration, errors, and memory warning behavior. |

## 6. Verification

- `frontmatter_lint` accepts this ADR.
- `tests/core/test_broadcast.py` verifies explicit broadcast behavior.
- Signature drift resolves section 5.1 to implementation symbols.

## 7. Consequences And Alternatives

Blocks keep full control over broadcast semantics, and invalid axis alignment
fails early. The cost is that block authors must call the helper explicitly.

Automatic engine broadcasting was rejected because shape match is not semantic
match. Leaving every block to hand-roll the pattern was rejected because it
would duplicate axis iteration and error handling.
