# Changelog Format

- Add entries under `[Unreleased]` for user-visible behavior, significant bug
  fixes, breaking changes, or plugin capability changes.
- Keep entries scoped, issue-linked, dated, and branch/session-attributable when
  workflow gate metadata exists.
- Do not use vague entries such as "misc", "update", or "fixes".
