# AI Developer Rules

These files modularize the active AI developer policy for SciEasy. `AGENTS.md`
remains the root entry point, and these files provide the detailed rule modules
that skills should point to.

Active rule modules:

- `root-policy.md`: project identity, architecture boundaries, and shortcut
  prohibitions.
- `gate-workflow.md`: the currently active `.workflow/gate.py` workflow.
- `personas.md`: AI developer personas and responsibility boundaries.
- `branch-pr-ci.md`: branch, commit, PR, and CI discipline.
- `docs-governance.md`: documentation and skill-pointer rules.
- `code-quality.md`: current lint, type, test, docs, frontend, and security
  checks.
- `protected-changes.md`: protected governance, workflow, and core paths.
- `hotfix.md`: live-debugging exception and retrospective gate handling.
- `runtime-parity.md`: canonical skill source and runtime mirror rules.
