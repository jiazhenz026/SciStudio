# src/scieasy/qa/AGENTS.md — QA infrastructure

## Scope

Quality assurance tooling: schemas (frontmatter, maintainers, audit-report, facts, tracker, governance, test-quality, classification, doc-schemas), audit modules (doc_drift, closure, trailer_lint, fact_drift, classification_lint), workflow gate v2 + per-stage validators (shadow mode), and supporting infrastructure for ADR-042/043/044 enforcement.

## Policy

- This subtree is owned by the QA Infrastructure Overhaul cascade (ADR-042/043/044).
- **🚫 No edits outside an active ADR-042/043/044 implementation phase.** Edits during phase work are bounded by the phase's TC list.
- All schemas use pydantic `model_config = ConfigDict(extra="forbid")` (ADR-042 §5.2).
- All audit modules MUST be importable as `python -m scieasy.qa.audit.<name>` and accept a `--paths` filter argument.
- Mutation score threshold for this subtree is **≥0.90** (tighter than root R3's 0.75) per ADR-042 §4.5.
- No silent fallbacks. Every degraded behavior MUST be a `TODO(#NNN)` with tracking issue.
- Schemas frozen during a phase; changes require a numbered Phase 1A/B/C/D sub-PR or an ADR errata.
